# Twinny — Sistemas Core

## Agent System

O Twinny tem um agente simples via Ollama:

1. Extensão VS Code ativa
2. Ollama local processa prompt
3. Resposta é exibida no webview

## Tool System

O Twinny não tem tools avançadas:
- Apenas chat básico
- Sem read/write/edit
- Sem bash
- Sem MCP

## Context Assembly

O contexto é mínimo:
- Histórico de conversa
- Sem @context system
- Sem RAG
- Sem per-directory rules

## Pontos Fortes

1. Simplicidade
2. Privacidade

## Limitações

1. Sem tools
2. Sem RAG
3. Sem compactação
