Wizards (10 guiados)

O que e

Os 10 wizards canonicos do XForge, cada um com fluxo interativo passo-a-passo, state machine, e persistencia em .xforge/wizards/<nome>/state.json. Compartilham o mesmo engine (skill wizard-engine).

Quando usar

  • Quando precisa de fluxo guiado para tarefa complexa (criar projeto, modelar dominio, auditar qualidade)
  • Quando quer consistencia entre multiplas execucoes do mesmo fluxo
  • Quando precisa de state resumable (parar e continuar depois)

Como usar

Cada wizard e' invocado via slash command no XForge Code CLI. A sintaxe canonica:

/<wizard-name> [opcoes]

O wizard faz perguntas em sequencia, valida cada resposta, e salva state. Pode ser interrompido e retomado sem perder progresso.

Parametros

Cada wizard tem seus proprios parametros. Veja a pagina especifica de cada wizard para detalhes. Em comum, todos aceitam:

ParametroTipoDescricao
--dry-runboolMostra plano sem executar
--verboseboolOutput detalhado de cada step
--quietboolSuprime output interativo (CI/CD)

Exemplos praticos

Cenario 1: Descobrir qual wizard usar

Contexto: voce nao sabe qual wizard e' apropriado para sua tarefa.

# Lista todos os 10 wizards
/wizard-list

# Output: /forge, /learn, /domain, /genius, /knowledge,
#         /governance, /release, /incident, /quality, /onboard

Output esperado: lista com descricao curta de cada wizard.

Edge case: nenhum. Comando read-only.

Cenario 2: Retomar wizard interrompido

Contexto: voce comecou /forge ontem, foi interrompido, quer continuar.

/forge --resume
# Carrega state.json e continua de onde parou

Output esperado: wizard no step onde voce parou.

Edge case: se state.json nao existe, wizard comeca do step 1.

Cenario 3: Cancelar wizard em andamento

Contexto: voce quer abortar wizard e voltar depois.

# Pressione Ctrl+C ou ESC durante o wizard
# State e salvo automaticamente em state.json

Output esperado: state salvo, mensagem de confirmacao.

Edge case: state salvo fica em .xforge/wizards/<name>/state.json.

Troubleshooting

Wizard nao inicia

Causa: skill wizard-engine nao carregada ou dependencia faltando.

Solucao: verifique se .kilo/skills/wizard-engine/SKILL.md existe e reinicie XForge Code.

State corrompido

Causa: state.json com JSON invalido ou arquivo truncado.

Solucao: delete .xforge/wizards/<name>/state.json e comece de novo.

Wizard travado em um step

Causa: input do usuario nao corresponde aos patterns esperados.

Solucao: responda com `skip` para aceitar default, ou `cancel` para abortar.

Lista Completa dos 10 Wizards

Total: 10 wizards. Clique para detalhes:

  • /forge - Wizard guiado para bootstrap completo de projeto XForge novo.
  • /learn - Wizard para absorver conhecimento de um dominio novo no template.
  • /domain - Wizard para modelagem de dominio via DDD.
  • /genius - Wizard para adicionar um novo genio ao Conselho dos 66.
  • /knowledge - Wizard para knowledge ingestion (CODE, DOCUMENTS, IDEAS, ANNOTATIONS, FEEDBACK).
  • /governance - Wizard para auditoria completa de governanca (RBAC, policies, LGPD, audit trail).
  • /release - Wizard para preparar release production-ready (version, changelog, tags, rollback plan).
  • /incident - Wizard para resposta estruturada a incidentes (5W2H, postmortem, lessons learned).
  • /quality - Wizard para auditoria de qualidade (tests, cobertura, SOLID, debt tecnico).
  • /onboard - Wizard para integrar novo dev ao time e ao projeto.
