19 - Changelog Detalhado (v50.x)

O que e

O Changelog Detalhado lista TODAS as versoes v50.x do XForge com descricao das mudancas, links para DRs e PRs, e impacto (breaking/major/minor/patch). Diferente do /CHANGELOG.md (que segue SemVer e e imutavel por DR-0087), esta pagina e editavel e detalhada. Para versoes v3.x (anteriores a v50), consulte /CHANGELOG.md.

Quando usar

  • Quando precisa saber o que mudou entre duas versoes (ex: v50.20.0 vs v50.31.0)
  • Quando quer entender quando uma feature foi adicionada (ex: quando surgiu o Linear MCP?)
  • Quando precisa identificar breaking changes antes de fazer upgrade
  • Quando esta escrevendo relatorio de auditoria de mudancas
  • NAO use para versoes v3.x - va em /CHANGELOG.md ou git log

Como usar

Cada entrada segue formato: versao, data, tipo (MAJOR/MINOR/PATCH), resumo, lista de DRs incluidos, breaking changes, link para release notes.

# Exemplo de uso:
git log --oneline v50.25.0..v50.33.0
# Para detalhes: leia este changelog a partir da versao v50.33.0 ate v50.25.0

Filtros: use Ctrl+F para buscar por BREAKING ou numero de DR especifico. Tags estao em ordem decrescente (mais recente primeiro).

Parametros

ParametroTipoDefaultDescricao
--deversaov50.0.0Versao inicial do range
--ateversaoHEADVersao final. Default = ultima tag
--tipostringtodosFiltra: major, minor, patch, ou todos
--drintnenhumBusca versao que contem DR especifico
--breaking-onlyboolfalseMostra apenas versoes com breaking changes
--jsonboolfalseOutput em JSON para integracao CI

Exemplos praticos

Cenario 1: Diff entre duas versoes

Contexto: Voce atualizou de v50.20.0 para v50.33.0 e quer saber o que mudou.

git log --oneline v50.20.0..v50.33.0

Output esperado:

v50.33.0 aside links corrigidos
v50.32.0 force-close details on load
v50.31.0 layout com aside esquerda
v50.30.0 aside sticky por categoria (broken - reverted in v50.29)
v50.29.0 revert v50.28 layout
v50.28.0 refactor layout (REVERTIDO)
v50.27.0 mojibake fix index
v50.26.0 mojibake fix 39 pages
v50.25.0 root cleanup DR-0208
v50.24.0 Slack MCP DR-0207
v50.23.0 Linear MCP DR-0206
v50.22.0 MarkItDown MCP DR-0205
v50.21.0 Brave Search MCP DR-0204
v50.20.0 Sequential Thinking MCP DR-0203

Edge case: Use --stat para ver numero de arquivos alterados em cada commit.

Cenario 2: Breaking changes recentes

Contexto: Voce precisa saber quais versoes exigem migracao manual.

# Filtrar changelog por breaking changes apenas

Output esperado:

v50.25.0 BREAKING: estrutura do root mudou. Scripts .py movidos para scripts/manual_gen/. Diretorios .opencode/, .kilo/node_modules/, etc deletadas.
v50.20.0 BREAKING: ordem de campos em INDEX.json mudou (applicabilityScope agora obrigatorio)

Edge case: Para BREAKING antes de upgrade, faca backup de .kilo/ e .xforge/ antes de fazer pull.

Cenario 3: Quando X feature foi adicionada

Contexto: Voce quer saber quando o Slack MCP foi adicionado.

grep -l 'Slack' docs/manual/19-changelog.html

Output esperado:

v50.24.0 - Slack MCP (DR-0207): OPTIONAL Node-based, env SLACK_BOT_TOKEN + SLACK_TEAM_ID, ~15 tools (canais, mensagens, reacoes, files)

Edge case: Para encontrar DR especifico, procure por DR-0XXX no changelog.

Cenario 4: Timeline por tipo

Contexto: Voce quer ver apenas minor/patch releases (sem majors que exigem migracao).

# Filtrar changelog por tipo minor ou patch

Output esperado:

v50.33.0 PATCH: aside link fixes
v50.32.0 PATCH: force-close details on load
v50.31.0 PATCH: layout aside + section cards
v50.30.0 PATCH: aside por categoria (broken)
v50.29.0 PATCH: revert v50.28
v50.27.0 PATCH: mojibake fix index
v50.26.0 PATCH: mojibake fix 39 pages

Edge case: PATCH releases sao sempre safe to upgrade. MINOR pode ter breaking - verifique.

Cenario 5: Buscar DR especifico

Contexto: Voce quer saber em qual versao o DR-0208 foi implementado.

grep 'DR-0208' docs/manual/19-changelog.html

Output esperado:

v50.25.0 MAJOR: DR-0208 (Root Cleanup). 11 .py scripts movidos para scripts/manual_gen/, deletadas 98MB de bloat (.opencode/, .kilo/node_modules/, reports/visual/), atualizada .gitignore, nova regra temp-files-organization.md

Edge case: Para detalhes do DR, va em .xforge/decisions/DR-0208-root-cleanup.md

Troubleshooting

Changelog nao atualiza automaticamente

Causa: Esta pagina e editavel manualmente, nao gerada de git log

Solucao: Para gerar de git log: git log v50.0.0..HEAD --oneline. Para sincronizar changelog, edite este arquivo apos cada release.

Versao nao aparece no changelog

Causa: Release foi feita direto em main sem tag (improvavel dado o pipeline)

Solucao: Verifique git tag --list v50.*. Se faltando, recriar tag ou adicionar entrada manualmente aqui.

Link para DR quebrado

Causa: DR foi removido ou movido para archive

Solucao: Veja .xforge/decisions/ADR-INDEX.md para lista atualizada. DRs antigos estao em .xforge/decisions/archive/.

Diff entre versoes muito grande

Causa: Span muito largo (ex: v50.0.0 vs HEAD)

Solucao: Reduza o range: changelog --de v50.25.0 --ate v50.33.0. Para spans maiores, use git log com --stat para ter nocao do tamanho.
