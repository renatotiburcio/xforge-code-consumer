Wizard /forge - Criar Projeto Completo (v50.6.0)

O que e

/forge new <nome> e' o wizard de bootstrap completo. Cria um projeto XForge do zero com scaffold .kilo/ + .xforge/ + docs/, scaffold da aplicacao (backend + frontend), auth seguro, packages canonicos, padroes de arquitetura, integracoes externas, e CI/CD. Total: 7 passos interativos.

Auto-detection de patterns e integracoes: atraves das flags --patterns e --integrations, o wizard detecta quais packages NuGet/npm adicionar, quais classes/arquivos gerar, e quais configuracoes em Program.cs. Se voce nao especifica, o wizard usa defaults seguros baseados no stack.

Auth padrao: ASP.NET Core Identity (mais seguro): hash PBKDF2 com salt, lockout apos 5 tentativas, 2FA opcional, confirmacao de email, recuperacao de senha. Para APIs puras, usa JWT com Identity (HS256/RS256).

Sub-comandos: /forge new <nome>, /forge list-templates, /forge resume, /forge validate-config, /forge upgrade.

Quando usar

  • Quando vai criar projeto NOVO do zero (backend + opcionalmente frontend)
  • Quando quer consistencia entre multiplos projetos do time
  • Quando precisa de setup rapido com packages canonicos XForge (Identity, MediatR, AutoMapper, FluentValidation)
  • Quando quer LGPD pre-configurado (Identity + consentimento + audit)
  • Quando quer DDD + CQRS estruturado (Domain/Application/Infrastructure)
  • Quando precisa integracoes externas (Stripe, Hangfire, SendGrid, etc)

Como usar

Sintaxe canonica:

/forge new <nome-projeto> [opcoes]

# Exemplo com todas as opcoes principais
/forge new MeuProjeto --stack dotnet --template enterprise-v2 --auth identity --patterns ddd cqrs mediator automapper fluentvalidation --integrations stripe hangfire sendgrid --lgpd true --database postgresql --ci github

Exemplos praticos (6 cenarios completos)

Cenario 1: ERP enterprise completo (Identity + DDD + CQRS + Stripe + Hangfire)

Contexto: Empresa brasileira precisa de sistema completo com pagamentos online, jobs em background, e arquitetura DDD. Time de 6 devs, LGPD obrigatorio.

/forge new MeuProjeto --stack dotnet --template enterprise-v2 \
  --auth identity \
  --patterns ddd cqrs mediator automapper fluentvalidation result-pattern ddd-aggregates \
  --integrations stripe hangfire sendgrid signalr \
  --error-handling global-middleware \
  --lgpd true --database postgresql --ci github

Output esperado (resumo - ver exemplo completo em wiz-forge-examples.html):

  • 184 arquivos criados em ~25 segundos
  • Identity: AddIdentity + AddEntityFrameworkStores + IdentitySeeder (admin inicial)
  • DDD: Domain/Application/Infrastructure com Aggregates + VOs + DomainEvents
  • CQRS: Commands + Queries + Handlers com MediatR + ValidationBehavior
  • Stripe: StripePaymentService + WebhookController
  • Hangfire: ProcessPaymentsJob (recurring daily) + SendWelcomeEmailJob
  • SignalR: NotificationHub com grupos (user-{id})
  • Global Error: ProblemDetails RFC 7807 + custom GlobalExceptionHandler

Exemplos de codigo completo (Program.cs + DDD + CQRS + Stripe + Hangfire + SignalR + Identity): wiz-forge-examples.html (Cenarios 1-7 com snippets copy-pasteable).

Cenario 2: API pura (sem UI) com JWT

Contexto: Backend-only REST API consumido por frontend separado (React/Angular). Sem UI de login, usa JWT.

/forge new MeuProjeto.Api --stack dotnet --template enterprise-v2   --auth jwt   --patterns cqrs mediator automapper fluentvalidation result-pattern   --integrations stripe hangfire   --database postgresql

Diferenca do Cenario 1:

  • Identity removido, usa apenas JwtBearer
  • Auth via /auth/login endpoint que retorna JWT token
  • Refresh tokens em /auth/refresh
  • Frontends consomem JWT no header Authorization: Bearer
  • Sem Razor Pages (Identity UI)

Edge case: JWT sem Identity significa gestao de usuarios manual. Use --auth identity --no-ui (identity backend + sem UI razor).

Troubleshooting

Wizard trava em stack detection

Causa: stack nao detectado, wizard espera input infinito.

Solucao: use --stack dotnet explicito ou verifique se tem *.csproj/package.json/pyproject.toml.

Package restore falha

Causa: nuget offline, npm offline, ou package indisponivel.

Solucao: dotnet restore manual; verifique se internet esta OK.

Identity migration falha

Causa: connection string invalida ou Postgres nao acessivel.

Solucao: verifique appsettings.json e rode dotnet ef database update manualmente.

Stripe webhook nao funciona

Causa: webhook secret nao configurado.

Solucao: configure STRIPE_WEBHOOK_SECRET em appsettings.Production.json e Stripe Dashboard.

Hangfire jobs nao rodam

Causa: server Hangfire nao foi iniciado ou DB nao tem schema Hangfire.

Solucao: verifique app.UseHangfireServer() em Program.cs. Primeira run cria schema automatico.

Template nao encontrado

Causa: nome invalido.

Solucao: /forge list-templates para ver disponiveis (default, enterprise-v2, startup-minimal, fiscal-br, saas-multi-tenant).
