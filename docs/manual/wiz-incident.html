Wizard /incident - Responder Incidente

O que e

/incident guia resposta a incidente: identifica tipo (seguranca, performance, dados), escala severidade, captura timeline, sugere mitigacao, gera postmortem template, e armazena em .xforge/incidents/. Integra com Slack/email para notificacoes.

Quando usar

  • Quando acontece incidente em producao
  • Quando precisa de postmortem estruturado
  • Quando quer lessons learned catalogadas

Como usar

Sintaxe canonica:

/incident --severity critical --type security --summary "XSS em /search exposto dados pessoais" --notify slack+email+anpd --timeline ./logs/xss-exploit.json

Para invocar: abra o XForge Code CLI e digite o comando. O wizard guia via Q&A interativo.

Exemplos praticos

Cenario 1: Incidente de seguranca critico (data breach)

Contexto: Hacker explorou XSS em /search e acessou dados pessoais. Incidente LGPD com obrigacao de notificacao ANPD em 72h.

/incident --severity critical --type security --summary "XSS em /search exposto dados pessoais" --notify slack+email+anpd --timeline ./logs/xss-exploit.json

Output esperado:

[incident] INC-2026-001 criado. CRITICAL.
[incident] Postmortem template gerado (LGPD-compliant).
[incident] Slack #security notificado.
[incident] Email dpo@meuprojeto.com enviado.
[incident] ANPD notification template preparado (prazo 72h).

[CONTINUIDADE - acoes obrigatorias]:
  1. Revisar postmortem gerado
  2. Confirmar impacto (quantos registros afetados)
  3. Preparar notificacao ANPD (template em .xforge/incidents/)
  4. Bloquear exploit (patch XSS)
  5. Documentar lessons learned -> DR-0202
  6. Se aplicavel: rodar /forge upgrade --security

Edge case: Critical incidents LGPD tem SLA rigido (notificacao ANPD em ate 72h).

Cenario 2: Performance degradada (API lenta)

Contexto: API latencia subiu 5x nas ultimas horas. p95 de 200ms para 1000ms.

/incident --severity high --type performance --summary "API /orders p95 1000ms (normal: 200ms)" --timeline ./grafana-snapshot.json

Output esperado:

[incident] INC-2026-002 criado. HIGH.
[incident] Timeline importada (47 eventos).
[incident] Sugestoes de mitigacao:
  - Verificar N+1 queries (EF Core)
  - Verificar conexoes de banco (connection pool)
  - Verificar cache (Redis se --integrations redis)
  - Verificar jobs Hangfire (fila longa?)
[incident] Postmortem template gerado.

Edge case: Use --timeline ./grafana-snapshot.json para correlacionar com metricas.

Cenario 3: Disponibilidade (API fora do ar)

Contexto: API retornando 503 ha 15 minutos. Health check falhando.

/incident --severity critical --type availability --summary "API retornando 503 ha 15min" --notify slack+email+pagerduty

Output esperado:

[incident] INC-2026-003 criado. CRITICAL.
[incident] Slack #ops-critical notificado.
[incident] PagerDuty incident criado.
[incident] Sugestoes de mitigacao:
  - Verificar health check endpoint (/health)
  - Verificar logs de erro (Serilog -> Seq/Elastic)
  - Verificar conexao com banco (PostgreSQL)
  - Verificar memoria/CPU do container (Docker stats)
  - Rollback: docker-compose down && docker-compose up -d --force-recreate

Edge case: Use --notify pagerduty para escalar automaticamente.
