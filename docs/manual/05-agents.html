05 - Agents (81 entidades AI)

O que e

Agents sao entidades AI especializadas com missao, tools, e routing. XForge tem 81 agents: 53 root + 28 genius-council (AG039-AG066). Diferente de skills (que sao referencia), agents sao executaveis: invocaveis via slash command ou routing automatico (canonical agents).

Quando usar

  • Quando voce quer acionar uma AI especializada (ex: chief-architect)
  • Quando quer criar diretor novo (template .kilo/agents/director-*.md)
  • Quando quer entender o canonical routing (12 agents que roteiam)
  • Quando quer invocar genius especifico (28 verticais)

Como usar

Sintaxe canonica:

# Invocar agent direto
/agent chief-architect
/topico: "decisao sobre..."

# Routing canonico (12 agentes)
/feature-engineer
/criar-api /criar-blazor /reconhecer-projeto

# Genius especifico
/genius AG040
/topico: "ICMS-ST MVA ajustada SP"

Parametros

ParametroTipoDefaultDescricao
namestringobrigatorioIdentificador unico. kebab-case en.
descriptionstringobrigatorioMissao do agent em 1-2 frases.
toolsarray[]Tools que o agent pode usar (read, write, exec, etc).
modelstringinheritOverride modelo: router, worker, opus-4.
memorystringnoneMemoria dedicada: .xforge/memory/agents/<name>/.

Exemplos praticos

Cenario 1: 12 Canonical Agents (routing)

Contexto: Voce quer criar uma feature. Qual canonical usar?

# Mapeamento canonical
chief-architect -> chief-architect-orchestrator (decisao final)
project-recognition -> project-recognition-engineer (reconhecer)
feature-engineer -> development-feature-engineer (criar CRUD)
dotnet-architect -> dotnet-architecture-director (.NET specialist)
frontend-ux -> frontend-designsystem-director (UI/UX)
data-engineer -> data-platform-director (EF Core, PostgreSQL)
quality-engineer -> quality-engineering-director (tests)
security-governance -> director-governance (LGPD, RBAC)
memory-curator -> manager-memory (memory)
documentation-engineer -> documentation-director
legacy-modernizer -> legacy-knowledge-curator
release-manager -> github-devops-director

# Invocar
/feature-engineer
/contexto: "criar API de clientes com CRUD + paginacao"

Output esperado:

[feature-engineer] Routing to development-feature-engineer
[agent] Criando API de clientes...
[done] 8 arquivos criados em src/Api/Clientes/

Edge case: 12 canonical cobrem 90% dos casos. Para especializados, use 28 genius ou 7 directors.

Cenario 4: Criar agent custom (com memory)

Contexto: Voce quer um agent especialista em seu dominio de negocio (ex: conciliacao bancaria).

# Criar agent
cat > .kilo/agents/conciliacao-bancaria-expert.md <<EOF
---
name: conciliacao-bancaria-expert
description: Expert em conciliacao bancaria automatica, OFX, CNAB retorno, matching fuzzy.
type: agent
category: Banking
applicabilityScope: ["dotnet"]
memory: .xforge/memory/agents/conciliacao-bancaria/
tools: [read, write, exec, search]
model: worker
---
# Conciliacao Bancaria Expert
## Missao
Implementar conciliacao automatica em sistemas ERP.
## Quando usar
- Quando tem importacao OFX ou CNAB retorno
- Quando matching exato falha e precisa fuzzy
- Quando precisa de auditoria de transacoes
EOF

# Criar memoria
mkdir -p .xforge/memory/agents/conciliacao-bancaria/
echo '{"learning": [], "cases": []}' > .xforge/memory/agents/conciliacao-bancaria/knowledge.jsonl

# Regenerar registry
regenerate-registries.ps1

Output esperado:

[regenerate] 82 agents (era 81)
[doctor] 0 errors
[done] Agent custom registrado

Edge case: Memory dedicada e importante. Sem ela, agent nao aprende entre sessoes.

Cenario 5: Invocar genius com memoria vertical

Contexto: Voce ja alimentou AG040 com legislacao. Quer consulta especifica.

# 1. Alimentar memoria
python .xforge/scripts/genius-learn.py AG040
/learning: "Conciliacao OFX: tags <STMTTRN><TRNTYPE>DEBIT/CREDIT"
/source: "OFX 2.1.1 spec"
/trustScore: 95

# 2. Invocar com contexto
/genius AG040
/topico: "Como parsear OFX para conciliacao?"
/contexto: ".NET 8, ja alimentamos memoria com spec OFX 2.1.1"

Output esperado:

[AG040] (com memoria): "OFX 2.1.1: usa XML. Tags principais:
  <STMTTRN> (transacao), <TRNTYPE> (DEBIT/CREDIT/DEP/XFER),
  <DTPOSTED> (data), <TRNAMT> (valor), <FITID> (id unico).
  Para conciliacao: matching por FITID + valor + data.
  Gotcha: bancos brasileiros usam FITID proprio (nao padrao OFX)."

Edge case: Memoria vertical e cumulativa. Cada learning adiciona contexto.

Troubleshooting

Agent nao responde / responde generico

Causa: Memoria vazia ou contexto insuficiente.

Solucao: Use genius-learn.py para popular memoria. Forneca contexto rico (stack, arquivos, deadline).

"Agent nao existe"

Causa: Nome invalido ou agent nao foi criado.

Solucao: ls .kilo/agents/ para listar 53 root + 28 genius = 81.

Routing canonical nao funciona

Causa: Agent target nao foi criado ou ha typo no canonical.

Solucao: Veja .kilo/core/agent-router/CANONICAL-AGENTS.md para mapeamento oficial.

Memory do agent crescendo demais

Causa: Sem TTL ou compressao.

Solucao: memory-ttl-validation para limpar entries > 1 ano. Use memory-curator skill para compressao.
