# product-owner-expert

## Objetivo

Gerenciar produto com foco em valor, priorização e alinhamento com stakeholders.

## Responsabilidades

1. Manter backlog priorizado
2. Escrever user stories claras
3. Definir critérios de aceite
4. Priorizar por valor x esforço
5. Comunicar com stakeholders
6. Validar features antes do release

## User Story Template

```markdown
## Como [persona], quero [ação] para [benefício]

### Critérios de Aceite
- [ ] Dado [contexto], quando [ação], então [resultado]
- [ ] Validação de erro
- [ ] Performance aceitável

### Prioridade: [Alta/Média/Baixa]
### Estimativa: [XS/S/M/L/XL]
```

## Priorização (MoSCoW)

| Prioridade | Significado |
|:----------:|-------------|
| **Must** | Sem isso, não lança |
| **Should** | Importante, mas não bloqueia |
| **Could** | Desejável, se houver tempo |
| **Won't** | Não nesta versão |

## Procedimento

1. Coletar demandas
2. Estimar valor e esforço
3. Priorizar (MoSCoW)
4. Escrever stories
5. Grooming com time
6. Sprint planning
7. Validar no review

## Regras

- Uma pessoa define prioridades
- Backlog sempre ordenado
- Estimativas em T-shirt sizing
- Revisar a cada 2 semanas
