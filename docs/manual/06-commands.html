06 - Commands (144 slash commands)

O que e

Commands sao slash commands invocados pelo usuario no XForge Code CLI. XForge tem 144 commands em 14 categorias. Diferente de skills (referencia) e agents (executaveis AI), commands sao workflows: combinam 1+ skills/agents/scripts em uma acao coesa.

Quando usar

  • Quando voce quer executar um workflow complexo (criar projeto, gerar knowledge, etc)
  • Quando voce quer descobrir comandos por categoria (registry.json)
  • Quando voce quer criar command custom (reutilizavel)
  • Quando voce quer encadear commands (composability)

Como usar

Sintaxe canonica:

# Slash command no XForge Code CLI
/xforge-init
/quickstart
/genius-council
/create-api
/memory-recover-context

# Com args
/quickstart --stack dotnet
/genius-council /topico: "..." /contexto: "..."

# Listar commands disponiveis
/xforge-help

Parametros

ParametroTipoDefaultDescricao
namestringobrigatorioIdentificador do command. kebab-case en, prefixo /.
descriptionstringobrigatorio1-2 frases. Aparece em /help.
agentstringrouterQual agent executa: router (rapido), worker (inteligente), ou nome especifico.
categorystringcoreDevelopment, Governance, Knowledge, Quality, etc.
argsobject{}Argumentos nomeados que o command aceita.

Exemplos praticos

Cenario 1: 15 commands de Development

Contexto: Voce quer criar feature, API, ou projeto.

# Principais commands de dev
/create
/create-api
/create-blazor
/create-blazor-component
/create-dotnet-project
/create-erp
/create-feature
/create-project
/develop
/engineer-initial-bootstrap
/feature-engineer
/forge
/improve
/prototype
/ship

Output esperado (exemplo /create-api):

/create-api
/recurso: "clientes"
/verbos: "GET POST PUT DELETE"
/autenticacao: "JWT"
# [router] Roteando para feature-engineer
# [agent] Criando API REST de clientes...
# [done] 8 arquivos em src/Api/Clientes/

Edge case: Commands de dev exigem contexto de stack. Stack auto-detected ou passado via --stack.

Cenario 2: 10 commands de Recognition

Contexto: Voce quer analisar projeto existente.

Cenario 4: 8 commands de Wizard

Contexto: Voce quer seguir um wizard (fluxo guiado) para tarefa complexa.

# Wizards
/forge       # criar projeto do zero
/learn       # aprender dominio novo
/domain      # modelar dominio de negocio
/genius      # adicionar genio ao conselho
/knowledge   # ingerir conhecimento novo
/governance  # revisar governanca
/release     # preparar release
/incident    # responder incidente
/quality     # auditar qualidade
/onboard     # onboard novo dev

Output esperado (exemplo /forge):

/forge
# [wizard] Bem-vindo. Vamos criar um projeto novo.
# [step 1/7] Qual stack? [dotnet, node, python, ...] (default: auto)
# [step 2/7] Qual dominio? [erp, blog, api, ...]
# [step 3/7] LGPD? (y/n)

Edge case: Wizards sao interativos. Para automacao, use commands individuais (criar-api, etc).

Cenario 5: Criar command custom

Contexto: Voce quer command que automatiza uma tarefa recorrente.

cat > .kilo/commands/meu-deploy.md <<EOF
---
name: meu-deploy
description: Deploy do meu projeto para staging com smoke tests
type: command
agent: worker
category: Custom
args:
  - name: ambiente
    type: string
    required: true
  - name: skip-tests
    type: bool
    default: false
---
# Meu Deploy
1. Build .NET
2. Run tests (unless --skip-tests)
3. Deploy to {{ ambiente }}
4. Smoke test endpoint /health
5. Notify Slack
EOF

regenerate-registries.ps1

Output esperado:

[regenerate] 145 commands (era 144)
[doctor] 0 errors
[done] Command custom registrado, invocavel como /meu-deploy

Edge case: Commands podem encadear outros. Use composability: /meu-deploy && /xforge-health-check.

Troubleshooting

Command nao encontrado

Causa: Nome errado ou command nao foi criado.

Solucao: ls .kilo/commands/ | head -30 para ver 144 disponiveis. /xforge-help lista os mais usados.

Command executa mas da erro

Causa: Args invalidos ou contexto insuficiente.

Solucao: Use /<command> --help (se implementado) ou veja o markdown do command em .kilo/commands/<name>.md.

Command nao registrado

Causa: Esqueca de rodar regenerate-registries apos criar.

Solucao: regenerate-registries.ps1 apos criar/modificar command.

Command muito lento

Causa: Agent pesado (worker) ou workflows em loop.

Solucao: Use agent=router para tarefas simples. Para workflows complexos, worker e necessario.
