10 - Extensions (MCPs + Wizards + Workflows + Automation)

O que e

Extensions sao 4 tipos de extensibilidade: MCPs (3 servers), Wizards (10), Workflows (20 YAMLs), Automation scripts (90+). Diferente de skills (referencia) e commands (slash), extensions sao integracoes externas e workflows estruturados.

Quando usar

  • Quando voce precisa de integracao externa (MCP para product/market/compliance)
  • Quando quer seguir um wizard guiado (10 wizards canonicos)
  • Quando quer criar workflow versionado (YAML)
  • Quando quer automatizar via scripts PowerShell/Python

Como usar

Sintaxe canonica:

# MCPs (3 servers)
.kilo/mcp/product-engineering/   # SDD/PRD/backlog auto-gerados
.kilo/mcp/market-intelligence/  # reviews, features, concorrentes
.kilo/mcp/compliance-intelligence/ # legislacao atualizada

# Wizards (10)
.kilo/wizards/forge, learn, domain, genius, knowledge, governance, release, incident, quality, onboard

# Workflows (20 YAMLs)
.kilo/workflows/W001-W019/

# Automation scripts (90+)
.kilo/automation/scripts/

Parametros

ParametroTipoDefaultDescricao
namestringobrigatorioIdentificador. kebab-case en.
typestringobrigatoriomcp, wizard, workflow, automation.
versionstring1.0.0SemVer. Incremental em breaking.
dependenciesarray[]Skills, agents, commands usados.

Exemplos praticos

Cenario 1: MCP product-engineering (SDD/PRD auto)

Contexto: Voce quer gerar SDD completo a partir de descricao de feature.

# Invocar tool MCP
mcp__product-engineering__generate_sdd
/feature: "criar modulo fiscal com NFe"
/stack: "dotnet"
/complexity: "L"

# Output: SDD v1.0 com 12 secoes
# - Metadata, Context, Personas, Requisitos, Arquitetura, Dados
# - API, UI/UX, Testes, Observabilidade, Seguranca, Entrega

Output esperado:

[product-engineering] SDD gerado
[SDD] Salvo em .xforge/analysis/<feature>/06-sdd.md
[done] 12 secoes, 2,500 tokens, 85% completeness

Edge case: SDD generator precisa de input rico (feature, stack, complexidade). Input vago gera SDD generico.

Cenario 2: Wizard /forge (criar projeto do zero)

Contexto: Voce quer seguir wizard para criar projeto novo com escolhas guiadas.

/forge
# [wizard] Bem-vindo. 7 passos.
# [1/7] Stack? [dotnet, node, python, ...]
# [2/7] Dominio? [erp, blog, api, ...]
# [3/7] LGPD? (y/n)
# [4/7] Time size? [<5, 5-20, >20]
# [5/7] Infra? [cloud, on-prem, hybrid]
# [6/7] Knowledge custom? (path)
# [7/7] Confirmacao + execucao
# [done] Projeto bootstrapado

Output esperado:

[forge] Wizard completo
[plan] .kilo/ + .xforge/ + docs/ criados
[plan] 12 knowledge entries pre-criadas
[plan] 3 DRs sugeridas
[done] Pronto para commit

Edge case: Wizards sao interativos. Para automacao, use commands individuais (criar-api, criar-projeto).

Cenario 3: Workflow versionado (W019 execution loop)

Contexto: Voce quer criar workflow custom para sua equipe.

cat > .xforge/workflows/W020-meu-workflow.yaml <<EOF
name: meu-workflow
description: Workflow customizado para deploy
version: 1.0.0
steps:
  - name: build
    type: command
    command: dotnet build
  - name: test
    type: command
    command: dotnet test
  - name: deploy
    type: command
    command: ./deploy.sh production
    when: "{{ steps.test.exitCode == 0 }}"
  - name: notify
    type: command
    command: ./notify-slack.sh "Deploy done"
triggers:
  - push:main
EOF

# Invocar
.\.xforge\workflows\run.ps1 W020

Output esperado:

[workflow] W020 iniciado
[step 1/4] build: OK (12s)
[step 2/4] test: OK (45s, 87 testes passaram)
[step 3/4] deploy: OK (30s)
[step 4/4] notify: OK
[done] Workflow completo em 87s

Edge case: Workflows com schema invalido sao rejeitados. Valide com workflow.schema.json.

Cenario 3 (alternativa): Automation scripts (doctor.ps1)

Contexto: Voce quer rodar quality gates manualmente.

.\.kilo\automation\scripts\doctor.ps1

# Output: 156 checks, 0 errors
# Gate 4 (manual canonico): 24/24 OK
# Gate 5 (content depth): 17/17 OK
# [done] CI green

Output esperado:

[doctor] 5 gates executados
[gate 1] required structure: 25 OK
[gate 2] registries: 17 OK
[gate 3] commands/agents/skills: 369 OK
[gate 4] manual canonico: 24 OK
[gate 5] content depth: 17 OK
[done] 0 errors, 1 warning, 452 OKs

Edge case: doctor.ps1 roda em segundos. Use em pre-commit/push hooks.

Cenario 4: MCP market-intelligence (analise de competidor)

Contexto: Voce quer extrair features de um concorrente para o seu PRD.

# Invocar tool
mcp__market-intelligence__extract_reviews
/url: "https://concurrent.com/reviews"
/limit: 100

# Output: reviews extraidos com NLP
# Top features: "integracao SAP", "mobile offline", "BI"
# Pain points: "lentidao", "preco alto", "suporte fraco"

Output esperado:

[market-intelligence] 100 reviews extraidos
[features] 8 top features com frequency %
[pain-points] 5 top com frequency %
[opportunity] 3 gaps para explorar
[done] Relatorio em .xforge/analysis/competitor-XXX/

Edge case: Web scraping pode falhar (rate limit, anti-bot). Use --delay e --rotate-user-agent.

Cenario 5: MCP compliance-intelligence (legislacao)

Contexto: Voce quer atualizar knowledge com legislacao 2025 (Reforma Tributaria).

# Invocar tool
mcp__compliance-intelligence__knowledge_append
/source: "EC 132/2023 + PL 368/2024"
/statement: "Reforma Tributaria 2025: IBS + CBS substitui ICMS + PIS/COFINS a partir 2026"
/domain: "fiscal"
/trustScore: 100
/keywords: [reforma, ibs, cbs, icms, pis, cofins]

# Output: knowledge entry criada
# + genius AG040 notificado para self-learning
# + INDEX.json atualizado
# + RAG reindex

Output esperado:

[compliance] Knowledge entry criada
[path] .xforge/knowledge/dominios/fiscal/reforma-tributaria-2025.md
[INDEX] 321 entries (era 320, +1)
[RAG] reindex 749 docs
[genius] AG040 notified
[done] Memoria do genio atualizada

Edge case: MCP valida trustScore. Fontes < 50 sao rejeitadas (DR-0034).

Troubleshooting

MCP tool nao encontrado

Causa: MCP nao inicializado ou tool name errado.

Solucao: ls .kilo/mcp/ para ver 3 servers. Cada server tem tools.json com lista de tools.

Wizard trava em loop

Causa: Resposta invalida em alguma etapa.

Solucao: Ctrl+C para abortar. Reveja o que voce respondeu. Wizards validam cada step.

Workflow falha em step 3 (deploy)

Causa: Step 2 (test) falhou OU deploy.sh nao tem permissao.

Solucao: Rode step 2 manual primeiro. Verifique permissoes do deploy.sh.

Script PowerShell nao roda

Causa: ExecutionPolicy.

Solucao: Set-ExecutionPolicy -Scope CurrentUser RemoteSigned. Ou use pwsh (PowerShell Core) com -File.
