# Canonical Skills

This is the official skill facade for XForge on KiloCode.

Existing `.kilo/skills/*` folders remain available. Canonical skills define the preferred capability vocabulary for routing.

## Canonical Set

| Canonical skill | Purpose |
| --- | --- |
| `project-recognition` | Map project structure, stack, risks and project DNA. |
| `memory-retrieval` | Recover relevant `.xforge` memory on demand. |
| `memory-update` | Persist important learning without polluting memory. |
| `command-routing` | Map user intent to public commands and internal routes. |
| `dotnet-architecture` | .NET architecture, CQRS, endpoints, DTOs and packages. |
| `aspnet-core-api` | ASP.NET Core APIs, endpoints, auth and middleware. |
| `blazor-tailwind` | Blazor UI and Tailwind design system work. |
| `efcore-postgresql` | EF Core with PostgreSQL, migrations and health checks. |
| `efcore-mysql` | EF Core with MySQL/Pomelo and database conventions. |
| `testing-unit` | Unit test design and execution. |
| `testing-integration` | Integration tests, Testcontainers and WireMock. |
| `testing-architecture` | Architecture tests and dependency rules. |
| `testing-e2e-visual` | Playwright, visual checks and accessibility. |
| `stryker-mutation` | Mutation testing and quality thresholds. |
| `performance-benchmark` | BenchmarkDotNet, NBomber and performance gates. |
| `security-review` | Security review and secure code checks. |
| `lgpd-review` | Privacy, LGPD and data protection checks. |
| `release-readiness` | Release gates, rollback, notes and final audit. |
| `documentation` | Manuals, changelog, docs HTML and training. |
| `legacy-analysis` | Legacy study, reverse engineering and modernization. |
| `knowledge-ingestion` | Document/source ingestion into knowledge packs. |
| `knowledge-curation` | Trust score, deduplication, promotion and decay. |
| `github-devops` | GitHub Actions, releases, CODEOWNERS and CI/CD. |
| `incident-response` | Incident analysis, playbooks and known errors. |
| `product-ux-review` | Product, usability and end-user audit. |

## Selection Rule

Select the smallest set of skills needed for the task. Prefer one to three skills. Avoid loading large knowledge files until the skill asks for them.
