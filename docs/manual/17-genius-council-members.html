17 - Genius Council: 66 Membros

O que e

O Conselho dos Genios do XForge agrega 66 especialistas virtuais em 8 dominios (Fundamentos Computacao, Engenharia Software, Linguagens, Web/Infra, IA, Produto, UX/UI, Seguranca/Cripto) + 3 tiers verticais brasileiros (Tributario, Contabil, Trabalhista, SUAS, Investimentos, DDD). Diferente de LLMs genericos, cada genio tem persona historica, dominio de expertise, e memoria dedicada em .xforge/memory/genius/AGXXX/knowledge.jsonl.

Localizacao: definicoes em .kilo/rules/02-genius-council-framework.md (38 originais). Verticais em .kilo/agents/genius-council/AGXXX-*.md (28 brasileiros). Memoria por genio em .xforge/memory/genius/.

Quando usar

  • Quando precisa de perspectiva especializada em decisao nao-trivial (DR-0033)
  • Quando quer acionar multiplas visoes (ex: Norman + Nielsen para UX)
  • Quando precisa de memoria vertical especifica (ex: AG040 Ricardo Alexandre para ICMS)
  • Quando quer evitar conflito de especialistas via Devil's Advocate (AG999)
  • NAO use para tarefas triviais (Loop sozinho e mais rapido)
  • NAO use para gerar codigo generico (LLM worker faz melhor)

Como usar

Sintaxe canonica via slash command /genius-council:

/genius-council

# Com topico explicito
/genius-council
/topico: "escolha entre AutoMapper e Mapster"
/contexto: ".NET 10, projeto enterprise, time de 50 devs"
/restricao: "open-source, brasileiro, ZERO custo de licenca"

# Acionar genio especifico (sem Conselho completo)
/genius AG032
/topico: "threat model para API REST que lida com dados pessoais"

# Adicionar novo genio (CREATE_GENIUS)
/genius-create-genius
/nome: "Grace Hopper"
/inspiracao: "Grace Hopper, pioneira COBOL"
/dominio: "Linguagens / Compiladores"

Para uso programatico em DR:

# Documentar no DR quais genios foram acionados
**Genios consultados** (3):
- AG010 Anders Hejlsberg (TypeScript evolution)
- AG007 Robert C. Martin (Clean Code)
- AG012 Linus Torvalds (realidade operacional)

# Atualizar memoria do genio
python .xforge/scripts/genius-learn.py AG010
/learning: "TypeScript 5.5 adicionou TypePredicate generics em 2025"

Parametros

ParametroTipoDefaultDescricao
/genius <AGXXX>stringvazioID do genio especifico (ex: AG001, AG040). Se vazio, Conselho completo.
/topico <text>stringobrigatorioTopico da decisao. Concreto, nao abstrato.
/contexto <text>stringvazioStack, time, restricao, deadline, etc.
/restricao <text>stringvazioO que NAO pode ser violado (LGPD, custo, stack).
/genius-create-geniusflagfalseAdiciona novo genio (CREATE_GENIUS template).
/learning <text>stringvazioAtualiza memoria do genio com novo conhecimento.

Exemplos praticos

Cenario 1: Conselho completo para decisao arquitetural

Contexto: Voce precisa decidir entre microsservicos e monolito modular para um ERP BR. Aciona Conselho.

/genius-council
/topico: "Monolito modular ou microsservicos para ERP BR de medio porte?"
/contexto: "Time de 8 devs, 3 modulos (fiscal, contabil, vendas), C#/.NET 8, PostgreSQL, 500 clientes ativos"
/restricao: "LGPD compliance, time pequeno nao aguenta 8 deploys/dia, infra simples (1 servidor + replicas)"

Output esperado:

[GCF] Conselho acionado: 5 genios relevantes
[AG002 von Neumann] (arquitetura): "Modular monolith escala ate 100 devs, microsservicos adiciona complexidade distribuida sem ganho."
[AG045 Martin Fowler] (Patterns of Enterprise): "MonolithFirst. Comece monolito, extraia microsservicos quando dor justificar."
[AG012 Linus Torvalds] (realidade): "Equipe de 8 nao tem bandwidth para microservices. Foque em fazer o monolito BEM."
[AG017 Bill Gates] (escala): "Microsoft Windows monolito escalou para 1B usuarios. Bom design interno > distribuicao prematura."
[AG999 Devil's Advocate] (7 perguntas):
  1. Por que? Time novo, escopo definido. Premature optimization.
  2. Alternativa melhor? Modular monolith. 4 deploys/dia, 1 banco, ACID preservado.
  3. Mais simples? Sim, monolito.
  4. Mais barato? 1 cluster vs N clusters.
  5. Necessidade ou moda? Moda. "Microsservicos" vendeu curso de 5k USD.
  6. 5 anos? Monolito modular pode virar microsservicos depois (Strangler Fig).
  7. Maior critica? "Voces vao criar distributed monolith pior que o monolito simples."
[5 Guardians] (architecture, simplicity, security, quality, documentation):
  - OK: Modular monolith
[Consenso] Decisao: Monolito modular com 3 modulos internos (Fiscal, Contabil, Vendas).
            Justificativa: Time pequeno, escopo definido, LGPD exige ACID forte, distribuicao prematura adiciona risco.
            Risco: 3 anos depois, monolito pode virar bola de neve. Mitigacao: Bounded contexts internos, pronto para Strangler Fig.

[DR] Criando .xforge/decisions/DR-0194-monolith-vs-microservices.md
[Status] draft (aguarda sua aprovacao)

Edge case: Se topico e' muito vago ("como melhorar codigo?"), Conselho retorna "especifique mais". Seja concreto.

Cenario 2: Acionar genio vertical (AG040 - Tributario)

Contexto: Implementando calculo de ICMS-ST para SP. Quer consulta especializada.

/genius AG040
/topico: "Calculo de ICMS-ST para produtos de revenda em SP (MVA ajustada)"
/contexto: ".NET 8, C#, ja temos a tabela MVA, mas precisamos da MVA ajustada para importacao"
/restricao: "Convênio 52/2017 (substituicao tributaria), decreto SP 61.779/2016"

Output esperado:

[AG040 Ricardo Alexandre] (Tributario/ICMS):
  "ICMS-ST em SP segue MVA (Margem Valor Agregado) original OU MVA ajustada.
   MVA ajustada: MVA_original * (1 - ALQ_ICMS_INTERNO / (1 - ALQ_ICMS_INTERESTADUAL))
   Exemplo: MVA 50%, ALQ_SP 18%, ALQ_origem 12%:
   MVA_ajustada = 50% * (1 - 0.18 / (1 - 0.12)) = 50% * (1 - 0.18/0.88) = 50% * 0.7955 = 39.77%
   Verificar: 1. Decreto SP 61.779/2016 art. 26-A, 2. Se produto tem beneficio fiscal, ajustar MVA.
   Gotcha: SP publica MVA ajustada anualmente. Use a do ano corrente (jan/2025)."

[Memoria] Atualizado em .xforge/memory/genius/AG040/knowledge.jsonl:
  - ID: K-AG040-2025-001
  - Statement: MVA ajustada SP 2025: formula MVA_orig * (1 - 0.18/(1-0.12))
  - Source: Convenio 52/2017 + Decreto SP 61.779/2016
  - TrustScore: 100

Edge case: AG040 consulta legislacao vigente. Se a legislacao mudou (ex: Reforma Tributaria IBS/CBS), ele sinaliza "verificar com AG040 novamente em 2026".

Cenario 3: Devil's Advocate (AG999) em decisao ja tomada

Contexto: Voce ja decidiu usar Mapster. Quer validar se a decisao ainda se sustenta.

/genius AG999
/topico: "Mapster vs AutoMapper em projeto novo .NET 8"
/contexto: "DR-0073 ja decidiu AutoMapper, mas Mapster tem performance melhor (3-5x em benchmark recente)"
/decisao: "Manter AutoMapper por consistencia com XForge golden rules"

Output esperado:

[AG999 Devil's Advocate]:
  1. Por que manter AutoMapper? Golden rule XForge (PREF-001).
  2. Alternativa melhor? Mapster 3-5x mais rapido. Mas para 99% dos casos, irrelevante.
  3. Mais simples? AutoMapper tem mais docs, mais exemplos, mais integracao com .NET DI.
  4. Mais barato? Ambos open-source. Custo zero.
  5. Necessidade ou moda? "Performance" como argumento pode ser FOMO.
  6. 5 anos? AutoMapper tem manutencao ativa. Mapster idem. Ambos duraveis.
  7. Maior critica? "Voce priorizou consistencia sobre performance. Em hot path (10k req/s), 3x de slowdown importa."

[Veredito]: Decisao defensavel. Custo de troca > ganho de performance. Manter AutoMapper.
            Se performance virar gargalo (medir!), reconsiderar Mapster.
[Recomendacao]: Documentar benchmark na DR-0073 (anexo).

Edge case: AG999 pode vetar decisao via Guardian Security, mas nao via Devil's Advocate. AG999 levanta objecoes, voce decide.

Troubleshooting

Genio nao responde / resposta generica

Causa: Memoria do genio esta vazia ou sem contexto especifico.

Solucao: python .xforge/scripts/genius-learn.py AG040 para popular memoria com casos brasileiros. Sem memoria, genio responde generico.

"Genio AG0XX nao existe"

Causa: ID invalido. Os 66 vao ate AG066 (Tier 3). AG067+ nao existe ate ser criado.

Solucao: Use ID valido. Veja lista em .kilo/rules/02-genius-council-framework.md secao 2.

"Conselho demora > 30s"

Causa: Topico muito amplo, acionou 10+ genios.

Solucao: Limite a 3-5 genios explicitos: /genius-council AG007 AG012 AG999 /topico: .... Para GCF completo, use 7 fases (reserva 2-5 min).

DR-XXXX nao atualiza memoria do genio

Causa: Self-learning loop nao rodou. DR tem memoriaImpact: true?

Solucao: Edite DR adicionando no frontmatter: memoriaImpact: true. Rode genius-learn.py explicitamente. Ou commit o DR - hook pos-commit chama genius-learn.
