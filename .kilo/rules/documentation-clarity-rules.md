# Documentation Clarity Rules

## Use SEMPRE
- Verbos no infinitivo: Criar, Validar, Calcular
- Numeros concretos: p95 < 300ms, max 1000 req/s
- IDs estaveis: RF-001, US-001, RN-001, ADR-001
- Referencias cruzadas: ver RF-005, ver ADR-0012
- Exemplos concretos: input -> output
- N/A explicito (nunca deixe em branco)

## NUNCA use
- Rapido / Lento sem metrica
- Grande / Pequeno sem numero
- Aproximadamente sem range
- Fazer o possivel (sem criterio)
- O sistema (use o nome concreto)
- Deve ser considerado (vago)
- Em breve / Logo (sem prazo)
- etc (liste tudo)

## Estrutura
- 1 topico por paragrafo
- Listas para enumeracao
- Tabelas para comparacao
- Diagramas para relacao
- Codigo para exemplo

## Validacao (Teste do Desenvolvedor Cego)
- Como fazer? -> passo-a-passo?
- Quando? -> condicao mensuravel?
- Quanto? -> numero?
- Onde? -> localizacao?
- Por que? -> justificativa?

Se 1 pergunta fica sem resposta, doc falhou.
