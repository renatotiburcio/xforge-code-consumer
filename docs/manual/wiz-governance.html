Wizard /governance - Auditar Governanca

O que e

/governance roda checks automaticos em todos os aspectos de governanca do projeto: RBAC permissions, LGPD compliance, policy violations, audit trail integrity, e reporta gaps com sugestoes de DR. Gera relatorio em human-readable, JSON, ou JUnit.

Quando usar

  • Quando vai publicar projeto em producao (auditoria final)
  • Quando LGPD precisa ser verificada
  • Quando quer garantir que policies estao sendo seguidas

Como usar

Sintaxe canonica:

/governance --scope all --severity low --format junit --output ./audit/

Para invocar: abra o XForge Code CLI e digite o comando. O wizard guia via Q&A interativo.

Parametros

ParametroTipoDefaultDescricaoObrigatorio?

Exemplos praticos

Cenario 1: Auditoria pre-producao (completa)

Contexto: Vai publicar MeuProjeto em producao. Quer auditoria completa antes do deploy.

/governance --scope all --severity low --format junit --output ./audit/

Output esperado:

[governance] 47 checks executados...
  [OK] RBAC: 3 roles (Admin, Manager, User) com permissoes corretas
  [OK] LGPD: DPO configurado, retencao 5 anos, consent flow ativo
  [OK] Policies: 12 policies ativas, 0 violacoes
  [OK] Audit trail: .xforge/audit/ com 23 entries
  [WARN] CORS: AllowAnyOrigin() em Development (OK para dev)
  [WARN] JWT: Secret < 32 chars (recomendado: openssl rand -hex 32)
  [OK] Tests: 87% coverage (target 85%)
  [OK] Build: 0 errors, 2 warnings

[governance] Relatorio: ./audit/governance-report.junit.xml
[governance] Resultado: 45 OK, 2 WARN, 0 FAIL. Pronto para producao.

Edge case: Use --strict em CI para falhar em qualquer warning.

Cenario 2: Auditoria LGPD apenas

Contexto: Precisa verificar LGPD compliance antes de auditoria externa.

/governance --scope lgpd --format json --output ./lgpd-report.json

Output esperado:

[governance] 12 checks LGPD...
  [OK] DPO: dpo@meuprojeto.com configurado
  [OK] Retencao: 5 anos (Identity + audit logs)
  [OK] Consent: Identity consent flow ativo
  [OK] Privacy by Design: .xforge/knowledge/compliance/ entries presentes
  [WARN] Cookie consent: nao implementado (recomendado para frontend)
  [OK] Data export: endpoint /api/me/export implementado
  [OK] Right to deletion: endpoint /api/me/delete implementado

[governance] Gap: cookie consent. Sugestao: DR-0201-cookie-consent.md

Edge case: LGPD exige retencao < 5 anos para dados pessoais. Identity config: 5 anos.

Cenario 3: Auto-fix de permissoes RBAC

Contexto: Time adicionou roles sem permissoes explicitas.

/governance --scope rbac --fix

Output esperado:

[governance] 8 permissoes faltantes detectadas...
  [FIX] Role "User": adicionado "ReadOwnProfile"
  [FIX] Role "Manager": adicionado "ReadAllProfiles"
  [FIX] Role "Admin": adicionado "DeleteUser"
  [FIX] Role "User": adicionado "UpdateOwnProfile"
  [MANUAL] Role "Manager": "DeleteUser" requer revisao manual (perigoso)

[governance] 6 permissoes auto-fixadas. 2 precisam revisao manual.

Edge case: Auto-fix NAO remove permissoes existentes (apenas adiciona faltantes).
