01 - Quickstart (5 min ate primeiro comando)

O que e

O Quickstart e o caminho minimo de 5 minutos para sair de zero ate o primeiro comando executado com sucesso no XForge. Ele cobre quatro passos: (1) instalar o template via git clone, (2) rodar o bootstrap (xforge-init.ps1) que cria a estrutura local, (3) validar com doctor.ps1, e (4) abrir no editor com a extensao XForge Code CLI instalada. Diferente de um simples git clone (que baixa arquivos passivamente), o XForge executa deteccao de stack, gera project-DNA, prepara RAG local e ativa 14 MCP servers configuraveis.

Quando usar

  • Quando voce esta conhecendo o XForge pela primeira vez
  • Quando precisa clonar projeto existente que ja usa XForge
  • Quando quer resetar .kilo/ + .xforge/ sem perder docs
  • NAO use para projetos legados grandes (> 500 arquivos)

Como usar

O fluxo canonico segue quatro comandos sequenciais. Cada um tem saida verificavel:

# PASSO 1 - Clonar o template
git clone https://github.com/renatotiburcio/xforge-enterprise-development-os.git meu-projeto
cd meu-projeto

# PASSO 2 - Bootstrap local
pwsh ./.kilo/automation/scripts/xforge-init.ps1

# PASSO 3 - Validar tudo
pwsh ./.kilo/automation/scripts/doctor.ps1

# PASSO 4 - Abrir no editor
code .
# No XForge Code CLI: /xforge-health-check

Sucesso = exit code 0 + ultimo bloco com contadores (ex: 54 agents scanned, 187 skill directories scanned).

Parametros

ParametroTipoDefaultDescricao
--stack stringautoForca deteccao de stack: dotnet, node, python, rust, go, java
--no-colorboolfalseDesativa ANSI colors no output (CI/CD)
--offlineboolfalseSkip remote fetches, usa apenas cache local
--minimalboolfalseSetup minimo (so .kilo/)
--dry-runboolfalseMostra plano sem executar
--template stringdefaultAplica template: fiscal-br, lgpd-strict, erp-base
--jsonboolfalseOutput em JSON estruturado

Exemplos praticos

Cenario 1: Aprendizado (5 min)

Contexto: Voce nunca usou XForge, quer ver rapido.

mkdir /tmp/aprende-xforge && cd /tmp/aprende-xforge
git clone --depth 1 https://github.com/renatotiburcio/xforge-enterprise-development-os.git .
pwsh ./.kilo/automation/scripts/xforge-init.ps1 --minimal

Output esperado:

[xforge-init] v50.25.0
[init] Modo minimal (so .kilo/)
[done] .kilo/agents/ (54), .kilo/skills/ (187)
[doctor] 0 errors. Healthy.

Edge case: Se git falhar sem internet, init sugere --offline.

Cenario 2: ERP BR (.NET + fiscal)

Contexto: Voce comeca ERP cliente brasileiro, quer LGPD+fiscal+contabil pre-config.

git clone https://github.com/renatotiburcio/xforge-enterprise-development-os.git erp-x
cd erp-x
pwsh ./.kilo/automation/scripts/xforge-init.ps1 --template fiscal-br --stack dotnet

Output esperado:

[template] fiscal-br (LGPD + fiscal + contabil)
[stack] .NET detectado
[init] skills: fiscal-tax-expert + lgpd-expert + sped-fiscal-expert
[doctor] 0 errors, 1 warning

Edge case: Se --stack dotnet sem *.sln, doctor falha. Use --stack auto.

Cenario 3: Air-gapped (sem internet)

Contexto: Servidor sem internet, instala via tarball pre-baixado.

# COM INTERNET:
git clone --depth 1 https://github.com/renatotiburcio/xforge-enterprise-development-os.git
tar -czf xforge-50.25.0.tar.gz xforge-enterprise-development-os/
# AIR-GAPPED:
tar -xzf xforge-50.25.0.tar.gz
cd xforge-enterprise-development-os
pwsh ./.kilo/automation/scripts/xforge-init.ps1 --offline

Output esperado:

[offline] Modo offline (sem fetches)
[init] .xforge/knowledge/ carregado de cache local
[init] .xforge/rag/ indexado (780 docs)
[doctor] 0 errors, 1 warning (TTL nao validado)

Edge case: Knowledge TTL expirado aparece como warning. Sincronize .xforge/knowledge/ manualmente.

Cenario 4: CI/CD headless

Contexto: Pipeline CI valida setup a cada PR, sem interacao.

# .github/workflows/ci.yml
jobs:
  xforge-health:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: sudo apt-get install -y powershell
      - run: pwsh ./.kilo/automation/scripts/xforge-init.ps1 --minimal --no-color
      - run: pwsh ./.kilo/automation/scripts/doctor.ps1 --json

Output esperado:

{"errors":0,"warnings":1,"oks":62}
CI green

Edge case: Use --no-color para evitar ANSI no log, --json para parsing.

Troubleshooting

PowerShell execution policy bloqueia

Causa: ExecutionPolicy default (Restricted) impede .ps1

Solucao: Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned (one-time). Em Linux/macOS use pwsh (ja tem permissao).

git ou pwsh nao encontrado

Causa: Dependencias nao instaladas ou fora do PATH

Solucao: Instale Git (git-scm.com, marque Add to PATH no installer). PowerShell 7+ (aka.ms/powershell).

xforge-init.ps1 nao encontrado

Causa: Voce nao esta na raiz do projeto

Solucao: cd ate pasta com .kilo/. Confirmar: Test-Path ./.kilo/automation/scripts/xforge-init.ps1

XForge Code CLI nao reconhece o projeto

Causa: Extensao nao instalada, versao antiga, ou workspace nao confiavel

Solucao: VS Code Extensions - XForge Code - Install (1.0+). Clique em Trust the authors. cat kilo.jsonc para verificar.

Doctor warning: docs/ 576 KB > 500 KB

Causa: Manual canonico rico em exemplos didaticos

Solucao: Warning informativo nao-bloqueante. Aceitar ate DR formal para atualizar limite.
