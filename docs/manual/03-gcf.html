03 - Conselho dos Genios (GCF, 66 membros)

O que e

O Conselho dos Genios (Genius Council Framework) e' o processo canonico de decisao do XForge desde 2026-06-17 (DR-0033). Agrega 66 especialistas virtuais em 8 dominios + 3 tiers brasileiros. Toda decisao nao-trivial passa por 7 fases: Discovery (Turing), Multi-perspectiva (3-8 genios), Devil's Advocate (AG999), 5 Guardians, Consensus (DR canonico), Documentation, Execution.

Quando usar

  • Decisao arquitetural (stack, pattern, framework)
  • Decisao de seguranca (auth, LGPD, threat model)
  • Decisao de produto (escopo, ROI, feature flag)
  • Refactor com impacto > 5 arquivos
  • NAO use para tarefas triviais (renomear, formatar)
  • NAO use para gerar codigo generico (LLM worker e' melhor)

Como usar

Sintaxe canonica (slash command):

/genius-council
/topico: "escolha entre AutoMapper e Mapster"
/contexto: ".NET 10, projeto enterprise, time 50 devs"
/restricao: "open-source, brasileiro, zero custo"

/genius AG040
/topico: "MVA ajustada SP para ICMS-ST"
/contexto: ".NET 8, produto revenda, 2026"
/restricao: "Convenio 52/2017"

Parametros

ParametroTipoDefaultDescricao
/genius <AGXXX>stringvazioID especifico. Se vazio, Conselho completo (5-8 genios).
/topicostringobrigatorioTopico da decisao (concreto).
/contextostringvazioStack, time, deadline.
/restricaostringvazioLGPD, custo, stack fixo.
/genius-create-geniusflagfalseAdiciona novo genio (CREATE_GENIUS).

Exemplos praticos

Cenario 1: Conselho completo (decisao arquitetural)

Contexto: Decidir monolito vs microsservicos para ERP BR medio.

/genius-council
/topico: "Monolito modular ou microsservicos para ERP BR medio porte?"
/contexto: "Time 8 devs, 3 modulos (fiscal, contabil, vendas), .NET 8, PostgreSQL, 500 clientes"
/restricao: "LGPD, time pequeno, infra simples"

Output esperado:

[GCF] 5 genios relevantes
[AG002 von Neumann] "Modular monolith escala ate 100 devs..."
[AG045 Fowler] "MonolithFirst. Extraia quando dor justificar..."
[AG012 Torvalds] "Equipe 8 nao tem bandwidth para microservices..."
[AG999 Devil's] 7 perguntas respondidas
[5 Guardians] OK
[Consenso] Monolito modular com 3 bounded contexts
[DR-0194] Criado, status: draft

Edge case: Topico vago ("como melhorar codigo?") retorna "especifique mais".

Cenario 2: Genio vertical (AG040 - Tributario/ICMS)

Contexto: Implementar MVA ajustada SP. Quer consulta especializada.

/genius AG040
/topico: "MVA ajustada SP para ICMS-ST em revenda"
/contexto: ".NET 8, produto nacional, importacao futura"
/restricao: "Convenio 52/2017, SP decreto 61.779/2016"

Output esperado:

[AG040 Ricardo Alexandre] (Tributario/ICMS):
  "MVA_ajustada = MVA_orig * (1 - ALQ_SP / (1 - ALQ_origem))
   Exemplo SP 18% origem 12%: 50% * (1 - 0.18/0.88) = 39.77%
   SP publica anualmente. Use 2025 vigente."

[Memoria] Atualizado com MVA ajustada SP 2025 + fonte

Edge case: Legislacao muda. AG040 sinaliza "verificar novamente em 2026" (Reforma Tributaria).

Cenario 3: Devil's Advocate (AG999) em decisao feita

Contexto: Decidiu AutoMapper. Quer validar.

/genius AG999
/topico: "AutoMapper vs Mapster - decisao ja tomada (DR-0073)"
/decisao: "Manter AutoMapper por golden rule XForge"

Output esperado:

[AG999 Devil's Advocate]:
  7 perguntas respondidas
  Veredito: Decisao defensavel. Custo de troca > ganho de performance.
  Recomendacao: Anexar benchmark na DR-0073.

Edge case: AG999 NAO veta. Guardian Security pode vetar (separado).

Cenario 4: Adicionar genio (Grace Hopper)

Contexto: Quer adicionar pioneira de compiladores.

/genius-create-genius
/nome: "Grace Hopper"
/inspiracao: "Grace Hopper, COBOL, primeiro compilador A-0 (1952)"
/dominio: "Linguagens / Compiladores"
/missao: "Lembrar que compiladores existem ha 70+ anos"
/especialidades: 5
/perguntas: 5
/responsaveis: 5

Output esperado:

[CREATE_GENIUS] OK
  [Criar] .kilo/agents/genius-council/AG067-grace-hopper.md
  [Criar] .xforge/memory/genius/AG067/knowledge.jsonl
  [Atualizar] count: 67
  [DR-0195] draft, aguarda aprovacao

Edge case: Cada genio novo DEVE ter DR formal. Self-learning memory obrigatorio.

Cenario 5: Atualizar memoria (self-learning)

Contexto: Nova Reforma Tributaria 2025. Atualizar AG040.

python .xforge/scripts/genius-learn.py AG040
/learning: "Reforma Tributaria 2025: IBS + CBS substitui ICMS + PIS/COFINS. Inicio 2026, transicao 2026-2033."
/source: "EC 132/2023 + PL 368/2024"
/trustScore: 90

Output esperado:

[genius-learn] AG040 atualizado
[knowledge.jsonl] Append K-AG040-2025-002
[contradiction-check] OK
[reindex] Knowledge router recarregado

Proxima consulta AG040 retornara memoria atualizada.

Edge case: Contradicao com memoria existente abre debate (AG999).

Troubleshooting

Genio nao responde / resposta generica

Causa: Memoria vazia ou sem contexto especifico.

Solucao: python .xforge/scripts/genius-learn.py AG040 para popular. Sem memoria, genio responde generico.

"Genio AG0XX nao existe"

Causa: ID invalido. Vao ate AG066 (Tier 3). AG067+ nao existe.

Solucao: Veja lista completa em 17-genius-council-members.

Conselho demora > 30s

Causa: Topico muito amplo, acionou 10+ genios.

Solucao: Limite a 3-5 genios: /genius-council AG007 AG012 AG999 /topico: ...

DR-XXXX nao atualiza memoria do genio

Causa: Self-learning loop nao rodou. DR tem memoriaImpact: true?

Solucao: Adicione memoriaImpact: true no frontmatter da DR. Rode genius-learn.py explicitamente.
