20 - Troubleshooting (problemas comuns e solucoes)

O que e

Esta pagina consolida os problemas mais frequentes ao usar o XForge, organizados por categoria (Setup, Execucao, MCP, RAG, Quality Gates, etc). Cada problema tem: sintoma literal (mensagem de erro), causa raiz, e solucao passo-a-passo. Para problemas especificos de dominio, consulte /support ou abra uma issue.

Quando usar

  • Quando voce recebe um erro e quer resolver rapidamente
  • Antes de abrir uma issue no GitHub (talvez a solucao ja esteja documentada aqui)
  • Quando um Quality Gate falha e voce nao sabe por que
  • Quando um MCP nao carrega ou retorna erro
  • NAO use para duvidas de uso de features - va em /commands ou /skills

Como usar

Problemas estao agrupados por categoria. Use Ctrl+F para buscar a mensagem de erro exata. Cada problema tem 3 secoes: Sintoma (o que voce ve), Causa (por que acontece), Solucao (como resolver).

# Workflow recomendado:
# 1. Copie a mensagem de erro
# 2. Ctrl+F aqui e cole
# 3. Ache o problema correspondente
# 4. Siga a solucao passo-a-passo

Se nenhum problema corresponder, rode doctor.ps1 e compartilhe o output ao abrir issue.

Parametros

ParametroTipoDefaultDescricao
--errostringnenhumMensagem de erro ou codigo a buscar
--categoriastringtodasFiltra: setup, execucao, mcp, rag, quality, deployment
--severidadestringtodasFiltra: baixa, media, alta, bloqueante
--auto-fixboolfalseMostra apenas problemas com auto-fix disponivel

Exemplos praticos

Cenario 1: PowerShell execution policy bloqueia .ps1

Contexto: Voce roda xforge-init.ps1 e recebe erro cannot be loaded because running scripts is disabled.

Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
# Re-abra terminal e rode novamente

Output esperado:

Politica RemoteSigned aplicada. Scripts locais rodam normalmente. Scripts de internet precisam ser assinados.

Edge case: NAO use Unrestricted (inseguro). Para CI/CD, use bypass policy por job.

Cenario 2: Doctor reporta mojibake

Contexto: Voce ve warning de mojibake em algum arquivo do .kilo/agents/.

python -c "content = open('.kilo/agents/.md', 'rb').read(); print('mojibake:', b'\xc3\x83' in content)"

Output esperado:

Se mojibake: rode temp/fix_mojibake_v2.py (cobre double e triple encoding).

Edge case: Bug comum em bulk replace - PowerShell -replace em modo Latin-1 + UTF-8 reencode = double/triple encoding.

Cenario 3: MCP nao aparece no XForge Code

Contexto: Voce configurou MCP no kilo.jsonc mas nao aparece nas tools.

# Verifique 3 coisas:
# 1. enabled: true (ou false para desabilitado)
# 2. command + args corretos
# 3. env vars definidas ou via env: prefixo
cat kilo.jsonc | grep -A 5 mcp

Output esperado:

Configuracao valida. Se ainda nao aparece, reinicie XForge Code CLI.

Edge case: MCPs NATIVE so precisam de enabled: false. MCPs OPTIONAL precisam de /mcp-lifecycle install.

Cenario 4: RAG nao retorna resultados

Contexto: Voce pesquisa por termo no RAG mas nao acha nada.

python .xforge/scripts/rag_local.py status

Output esperado:

Se stale=true: rode python .xforge/scripts/rag_local.py reindex. Se count=0: verifique .xforge/knowledge/INDEX.json.

Edge case: RAG busca por tokens (palavras), nao por semantica. Use termos exatos.

Cenario 5: Quality Gate pre-push falha

Contexto: Voce faz git push e recebe pre-push check failed.

# Rode localmente para ver detalhes
pwsh ./.kilo/automation/scripts/pre-push.ps1

Output esperado:

Mostra qual gate falhou (tests, coverage, lint, security). Resolva o erro especifico antes de tentar push.

Edge case: Para pular pre-push (NAO recomendado): git push --no-verify

Cenario 6: Size da pasta docs/ excede 500KB

Contexto: Doctor avisa docs/ tem 602 KB (limite 500 KB).

Get-ChildItem docs/manual/*.html | Sort-Object Length -Descending | Select-Object -First 5

Output esperado:

Top 5 paginas mais pesadas. Geralmente sao wizards (wiz-forge ~20KB) ou pages com muitos exemplos.

Edge case: Warning NAO-bloqueante. Para reduzir: comprimir exemplos ou aumentar limite em check-manual.ps1.

Cenario 7: Build error CS0246 tipo nao encontrado

Contexto: Voce esta em projeto .NET e recebe erro de tipo nao encontrado.

# Auto-fix SH-001: detectar e adicionar import
pwsh ./.kilo/automation/scripts/self-heal.ps1

Output esperado:

Auto-healing corrige 12 erros comuns (SH-001 a SH-012). Para erros nao listados, abra issue.

Edge case: Self-healing roda em pre-commit. Se erro aparece em build, talvez nao foi auto-fixado.

Troubleshooting

Nenhum dos problemas corresponde ao meu erro

Causa: Erro especifico do seu ambiente ou projeto

Solucao: 1. Rode doctor.ps1 2. Copie output completo 3. Abra issue no GitHub com: erro + doctor output + passos para reproduzir

Problema resolvido mas nao consigo atualizar a pagina

Causa: Cache do navegador servindo versao antiga

Solucao: Ctrl+Shift+R (hard reload) ou DevTools > Network > Disable cache. Para forcar refresh global, mude ?v=50.2.5 para ?v=50.2.6 em style.css/script.js

Script funciona em minha maquina mas nao em CI

Causa: Diferenca de OS, versao PowerShell, ou paths relativos

Solucao: Use paths absolutos, normalize line endings (CRLF no Windows, LF no Linux). Para PowerShell, use 7+ com PowerShell Core (pwsh)

Erro intermitente (as vezes funciona, as vezes nao)

Causa: Race condition, cache stale, ou rate limiting de API externa

Solucao: Adicione retry logic com backoff exponencial. Para RAG, use .xforge/scripts/rag_local.py (local, sem rate limit).
