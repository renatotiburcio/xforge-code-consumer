Wizard /knowledge - Ingerir Conhecimento

O que e

/knowledge orquestra 5 tipos de ingestao: CODE (snippets), DOCUMENTS (arquivos), IDEAS (notas), ANNOTATIONS (comentarios), FEEDBACK (code review). Gera knowledge entries com trust score automatico. Indexa no RAG via rag_local.py reindex.

Quando usar

  • Quando quer adicionar knowledge entry especifica
  • Quando tem feedback de code review para registrar
  • Quando quer consolidar anotacoes de codigo em knowledge

Como usar

Sintaxe canonica:

/knowledge --type FEEDBACK --source "PR-1234: usar XUnit nao NUnit" --domain backend

Para invocar: abra o XForge Code CLI e digite o comando. O wizard guia via Q&A interativo.

Parametros

ParametroTipoDefaultDescricaoObrigatorio?
--typestringobrigatorioCODE, DOCUMENTS, IDEAS, ANNOTATIONS, FEEDBACKSim
--sourcestringobrigatorioPath, URL, ou texto inline (- para stdin)Sim
--domainstringautofiscal, lgpd, erp, backend, etcNao
--trust-scoreintauto0-100 (override auto-detection)Nao
--outputstring.xforge/knowledge/<domain>/Diretorio de saidaNao
--max-entriesint50Maximo de knowledge entries a gerarNao
--dry-runboolfalseMostra plano sem criar arquivosNao

Exemplos praticos

Cenario 1: Ingerir de code review (PR feedback)

Contexto: Voce recebeu feedback sobre um PR. Quer consolidar em knowledge.

/knowledge --type FEEDBACK --source "PR-1234: usar XUnit nao NUnit" --domain backend

Output esperado:

[knowledge] Source: inline text (45 chars)
[knowledge] Detectando: dominio backend, trust 85 (feedback de code review)
[knowledge] Gerando 1 knowledge entry:
  [+] .xforge/knowledge/compliance/code-review/xunit-only.md (trust: 85)
[knowledge] Criando 1 DR:
  [+] DR-0203-xunit-only-decision (status: draft, basado em feedback)
[knowledge] RAG reindexed (749 docs)

Edge case: FEEDBACK com trust >=80 entra automaticamente no rule registry.

Cenario 2: Ingerir de URL publica (docs Microsoft)

Contexto: Quer aprender sobre EF Core a partir de docs oficiais.

/knowledge --source https://learn.microsoft.com/ef-core/dbcontext-configuration --stack dotnet --domain backend --trust-score 95

Output esperado:

[knowledge] URL: https://learn.microsoft.com/ef-core/dbcontext-configuration
[knowledge] Fetching (47 requests, delay 1000ms)...
[knowledge] Extraindo texto de 23 paginas...
[knowledge] Detectando: dominio backend, trust 95 (Microsoft oficial)
[knowledge] Gerando 5 knowledge entries:
  [+] .xforge/knowledge/packs/ef-core/dbcontext-config.md
  [+] .xforge/knowledge/padroes/data-access/ef-core-patterns.md
  [+] .xforge/knowledge/glossario/ef-core-terms.md
  [+] .xforge/knowledge/compliance/ef-core-migrations.md
  [+] .xforge/knowledge/compliance/connection-strings.md
[knowledge] Criando 1 skill canonica:
  [+] .kilo/skills/ef-core-canon/SKILL.md
[knowledge] RAG reindexed (754 docs)

Edge case: Rate-limit Microsoft (429). Use --delay 3000 ou rode em horario de baixo trafico.

Cenario 3: Ingerir de codigo + comentarios (ANNOTATIONS)

Contexto: Voce tem um projeto legado com comentarios valiosos espalhados.

/knowledge --type ANNOTATIONS --source ./src/legacy/ --domain backend --keywords "refactor,pattern,tradeoff"

Output esperado:

[knowledge] Scanning 47 arquivos em src/legacy/
[knowledge] Encontrados 23 comentarios com keywords relevantes
[knowledge] Filtrando: 8 comentarios com profundidade (> 3 linhas)
[knowledge] Gerando 3 knowledge entries:
  [+] .xforge/knowledge/padroes/refactoring/legacy-service-extraction.md
  [+] .xforge/knowledge/padroes/architecture/anemic-domain.md
  [+] .xforge/knowledge/padroes/data-access/repository-split.md
[knowledge] RAG reindexed (757 docs)

Edge case: Anotacoes com TODO/FIXME sao ignorados (muito transient).
