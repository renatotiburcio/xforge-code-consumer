Wizard /visual-investigate - Investigacao Visual Automatica

O que e

/visual-investigate e' o wizard de investigacao visual automatica. Usa Playwright para navegar paginas web, capturar screenshots, validar layouts, testar responsividade (multi-viewport), dark mode, dropdowns, fluxos de usuario, e acessibilidade (axe-core). Tudo automatico — sem intervencao humana.

Sub-comandos: /visual-investigate run, /visual-investigate compare, /visual-investigate a11y, /visual-investigate report.

Quando usar

  • Quando quer investigar todas as paginas de um app web automaticamente
  • Quando precisa validar layout em multiplos viewports (mobile, tablet, desktop)
  • Quando quer testar dark mode, dropdowns, navegacao
  • Quando precisa de auditoria de acessibilidade (WCAG)
  • Quando quer detectar regressao visual (comparar com baseline)

Como usar

Sintaxe canonica:

# Investigacao completa (screenshots + responsividade + dark mode)
/visual-investigate --url http://localhost:5001 --pages all --viewports desktop,mobile --dark-mode

# Apenas acessibilidade (axe-core)
/visual-investigate --url http://localhost:5001 --accessibility --wcag-level AA

# Comparacao com baseline (regressao visual)
/visual-investigate --url http://localhost:5001 --compare --baseline reports/baseline/

# Multi-browser
/visual-investigate --url http://localhost:5001 --browsers chromium,firefox,webkit

Parametros

ParametroTipoDefaultDescricao
--urlstringhttp://localhost:5001URL base do app
--pagesstringallPaginas: all, /path, ou lista separada por virgula
--viewportsarraydesktopmobile (375x667), tablet (768x1024), desktop (1280x720), wide (1920x1080)
--browsersarraychromiumchromium, firefox, webkit
--accessibilityboolfalseRodar testes axe-core (WCAG)
--wcag-levelstringAAA, AA, AAA
--compareboolfalseComparar com baseline (regressao visual)
--baselinestringreports/baseline/Diretorio de screenshots baseline
--outputstringreports/visual/Diretorio de saida
--dark-modebooltrueTestar toggle dark mode
--flowsarray[]Fluxos: login, navigate, search, form
--timeoutint60000Timeout por teste (ms)

Cenario 2: Auditoria de acessibilidade (WCAG AA)

Contexto: LGPD exige acessibilidade. Quer validar todas as paginas contra WCAG 2.1 Level AA.

/visual-investigate --url http://localhost:5001 --accessibility --wcag-level AA --pages all

Output esperado:

[a11y] Running axe-core on http://localhost:5001...
[a11y] Pages tested: 47
[a11y] 0 critical violations
[a11y] 2 serious violations:
  - Color contrast ratio < 4.5:1 on .nav-link (foreground #0ea5e9 on #fff)
  - Missing aria-label on #darkBtn
[a11y] 4 minor violations:
  - Missing lang attribute on 1 page
  - 3 images without alt text
[a11y] Report: reports/a11y/axe-report.json

[CONTINUIDADE]:
  1. Corrigir contraste: .nav-link { color: #0284c7; } (WCAG AA: 4.5:1)
  2. Adicionar aria-label: <button id="darkBtn" aria-label="Toggle dark mode">
  3. Adicionar alt text em imagens
  4. Rodar /visual-investigate --accessibility novamente para validar

Edge case: WCAG AAA e' mais restritivo; use --wcag-level AAA para compliance total.

Cenario 3: Regressao visual (comparar com baseline)

Contexto: Voce fez mudancas no CSS. Quer verificar se alguma pagina mudou visualmente.

# 1. Gerar baseline (antes das mudancas)
/visual-investigate --url http://localhost:5001 --pages all --output reports/baseline/

# 2. Fazer mudancas no CSS...

# 3. Comparar com baseline
/visual-investigate --url http://localhost:5001 --pages all --compare --baseline reports/baseline/ --output reports/diff/

Output esperado:

[visual-investigate] Comparing with baseline...
[visual-investigate] Pages compared: 47
[visual-investigate] 45 pages: identical
[visual-investigate] 2 pages: DIFF detected
  - /manual/02-architecture.html: header nav shifted 2px right
  - /manual/07-rules.html: table width changed (920px -> 940px)

[visual-investigate] Diff images saved to reports/diff/
  - /manual/02-architecture.html-diff.png (before | after | diff)
  - /manual/07-rules.html-diff.png (before | after | diff)

[visual-investigate] Report: reports/diff/regression-report.md

Edge case: Se baseline nao existe, wizard gera automaticamente na primeira execucao.

Contexto: Quer testar o fluxo completo: login -> navegar -> criar entidade -> logout.

/visual-investigate --url http://localhost:5001 --flows "login,navigate,create" --browsers chromium,firefox

Troubleshooting

App nao encontrado (connection refused)

Causa: App nao esta rodando na URL especificada.

Solucao: Inicie o app primeiro: dotnet run (ou npm run dev). Verifique com curl http://localhost:5001.

Playwright nao instalado

Causa: npx playwright retorna erro.

Solucao: .\.kilo\automation\scripts\e2e\install-playwright.ps1 ou npm install -D @playwright/test && npx playwright install.

Timeout em pagina lenta

Causa: Pagina demora mais que 60s para carregar.

Solucao: Aumente timeout: --timeout 120000. Ou use --wait-until domcontentloaded (mais rapido que networkidle).

Elemento nao encontrado

Causa: Seletor CSS nao corresponde ao DOM atual.

Solucao: Use npx playwright codegen http://localhost:5001 para gerar seletores interativamente.

Baseline nao existe

Causa: Primeira execucao com --compare sem baseline.

Solucao: Wizard gera baseline automaticamente na primeira execucao. Use --output reports/baseline/ para salvar.
