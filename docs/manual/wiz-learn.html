Wizard /learn - Absorver Documentacao Existente

O que e

/learn orquestra knowledge ingestion de 5 fontes: CODE (snippets), DOCUMENTS (arquivos), IDEAS (notas), ANNOTATIONS (comentarios), FEEDBACK (code review). Le, classifica, gera knowledge entries com trust score, e indexa no RAG. Suporta continuar projeto (absorve de pasta pre-existente) ou criar do zero (absorve specs/repos antes de gerar codigo).

Sub-comandos: /learn source <path|url>, /learn feedback <pr-id>, /learn resume, /learn list-sources.

Quando usar

  • Apos /forge, para absorver documentacao pre-existente do projeto (specs, RFCs, decisoes)
  • Quando ha pasta legado-docs/ com documentacao dispersa
  • Quando precisa consolidar feedback de code reviews em knowledge
  • Quando quer aprender um framework novo a partir de docs oficiais

Como usar

Sintaxe canonica:

/learn source <path|url|texto> [opcoes]

# Exemplo com caminho
/learn source ./legado-docs/ --domain app --format auto --trust-score 80

# Exemplo com URL
/learn source https://learn.microsoft.com/ef-core --stack dotnet

Parametros

ParametroTipoDefaultDescricaoObrigatorio?

Troubleshooting

Wizard nao detecta formato

Causa: extensao nao suportada (.docx, .xlsx).

Solucao: converta para .md ou .pdf antes; pandoc file.docx -o file.md.

Trust score muito baixo (todos < 50)

Causa: fontes nao confiaveis (sem autor, sem data).

Solucao: use --trust-score 80 explicito (override) ou forneca docs oficiais.

Knowledge duplicada

Causa: entry similar ja existe.

Solucao: wizard abre debate (AG999); aceite ou rejeite duplicatas.

RAG nao atualiza

Causa: rag_local.py reindex nao foi chamado.

Solucao: wizard ja faz reindex automaticamente; verifique .xforge/rag/manifest.json.

PDF nao parseia

Causa: PDF escaneado (sem texto).

Solucao: use --extract-pdf (requer tesseract OCR) ou converta antes.
