09 - Decisoes (DRs, 25 ativos)

O que e

Decision Records (DRs) sao a fonte canonica de decisoes arquiteturais em .xforge/decisions/. XForge tem 25 DRs ativos + 52 em archive. Cada DR segue template 3-fases (Problema, Decisao, Riscos) ou 7-fases (alto impacto) per DR-0087. DRs sao imutaveis: alteracoes exigem NOVA DR que referencia a anterior (release-immutability).

Quando usar

  • Quando voce precisa tomar decisao arquitetural nao-trivial
  • Quando quer consultar decisao canonica (ex: "usamos X ou Y?")
  • Quando quer criar DR nova (apos GCF)
  • Quando quer auditar decisoes deprecated (archive)
  • NAO use para tarefas triviais (decisao cabe no commit message)

Como usar

Sintaxe canonica (via GCF ou manual):

# Criar DR via GCF (recomendado)
/genius-council
/topico: "..."
# Resultado: DR-XXXX-titulo.md criado em .xforge/decisions/

# Criar DR manual
cat > .xforge/decisions/DR-NNNN-titulo.md <<EOF
# DR-NNNN - Titulo
**Status**: draft
**Date**: YYYY-MM-DD
**Type**: architecture | process | pattern
## Problema
...
## Contexto
...
## Decisao
...
## Riscos
...
EOF

# Ver indice
cat .xforge/decisions/ADR-INDEX.md

Parametros

ParametroTipoDefaultDescricao
IDstringDR-NNNNNumero sequencial. NNNN segue ranges canonicos.
Statusstringdraftdraft, review, approved, deprecated.
Typestringarchitecturearchitecture, process, pattern, security, domain.
DatestringYYYY-MM-DDData da criacao.
ApprovalstringvazioQuem aprovou (humano + data).

Exemplos praticos

Cenario 1: Ler DR canonica (DR-0033 GCF)

Contexto: Voce quer entender o processo de decisao do XForge.

cat .xforge/decisions/DR-0033-genius-council-framework.md

Output esperado (excerpt):

# DR-0033 - Genius Council Framework (GCF)
**Status**: approved
**Date**: 2026-06-17
## Problema
Sistemas complexos falham por decisoes sem rastreabilidade.
## Decisao
Toda decisao nao-trivial passa pelo Conselho (7 fases).
## Consequencias
Positivas: rastreabilidade, qualidade, alinhamento.
Negativas: ceremony, tempo.

Edge case: DRs approved sao IMUTAVEIS (DR-0087). Mudancas => nova DR.

Cenario 2: Criar DR via GCF (recomendado)

Contexto: Decidir monolito vs microsservicos. Aciona GCF.

/genius-council
/topico: "Monolito modular ou microsservicos para ERP BR medio porte"
/contexto: "Time 8 devs, 3 modulos, .NET 8, PostgreSQL"
/restricao: "LGPD, time pequeno, infra simples"

Output esperado:

[GCF] 5 genios acionados
[AG002, AG045, AG012, AG017, AG999]
[Consenso] Monolito modular com 3 bounded contexts
[DR] Criando .xforge/decisions/DR-0194-monolith-vs-microservices.md
[Status] draft (aguarda sua aprovacao)

Edge case: GCF demora 2-5 min para decisoes L/XL. Para S/M, use decisao direta sem GCF.

Cenario 3: DR categories (ranges canonicos)

Contexto: Voce quer saber qual numero usar para nova DR.

cat .xforge/decisions/ADR-INDEX.md | Select-String "Range" -SimpleMatch

# Ranges canonicos
3300-3399: GCF, meta-regras (DR-0033, DR-0180, DR-0181, DR-0182)
3400-3499: Multi-stack, arquitetura (DR-0034, 0035, 0044, 0052, 0053, 0055)
3600-3699: Engine, sistemas (DR-0036, 0037, 0043, 0047, 0048)
3800-3899: Quality, testes (DR-0038, 0039, 0050)
4000-4099: AI, provedores (DR-0040, 0046, 0054)
4100-4199: Padroes, convencoes (DR-0041, 0042, 0045, 0049, 0056)
5100-5199: Dominios de negocio (DR-0051)
6700-6799: Releases v3.5-v3.7 (DR-0064 a 0074)
7500-7599: v3.7.1 compat (DR-0075 a 0079)
8000-8099: v3.7.2 cleanup (DR-0080)
9000+: Reservado para futuro

Output esperado:

Para DR nova em "qualidade de codigo" -> use 38XX
Para DR nova em "AI" -> use 40XX
Para DR nova em "release v50" -> use 80XX+

Edge case: Se usar numero fora do range canonico, ADR-INDEX fica fora de ordem. Mantenha ranges.

Cenario 4: DR template (3-fases vs 7-fases)

Contexto: Voce quer criar DR mas nao sabe qual template usar.

# 3-fases (default, 90% das decisoes) - DR-0087
# Problema: 1 paragrafo
# Contexto: 2-5 paragrafos
# Decisao: opcao escolhida + justificativa
# Riscos: tabela

# 7-fases (alto impacto) - DR-0033
# 1. Discovery
# 2. Multi-perspectiva
# 3. Debate (AG999)
# 4. Validation (5 Guardians)
# 5. Consensus
# 6. Documentation
# 7. Execution

# Quando usar 7-fases:
# - Decisoes arquiteturais multi-modulo
# - Breaking changes
# - Incident response
# - Sprint planning
# - Multi-team impact

Output esperado (3-fases snippet):

## Problema
[1 paragrafo descrevendo o que precisa ser decidido]

## Contexto
[2-5 paragrafos com situacao, restricoes, alternativas consideradas]

## Decisao
[opcao escolhida + 2-3 frases de justificativa]

## Riscos
| Risco | Probabilidade | Mitigacao |
| ... | ... | ... |

Edge case: 7-fases eh para GCF completo. 3-fases eh para decisao direta com Conselho minimo.

Cenario 5: DR deprecated (imutabilidade)

Contexto: Voce quer consultar DR antiga que foi superada.

ls .xforge/decisions/archive/ | head -10
# DR-0001-stack-original.md
# DR-0002-monolith-first.md (deprecated por DR-0194)
# ...

# DR-0002 vs DR-0194
# DR-0002 (v1, deprecated): "Use sempre monolito"
# DR-0194 (v2, approved): "Modular monolith com bounded contexts"
# DR-0194 substitui DR-0002 (referenciado em Related)

Output esperado (relacionamento):

**Related**: DR-0002 (deprecated, now superseded)
**Status**: approved
**Date**: 2026-06-21
**Supersedes**: DR-0002

Edge case: DRs deprecated NAO sao apagadas. Archive preserva audit trail.

Troubleshooting

DR nao encontrada

Causa: ID errado ou caminho errado.

Solucao: Get-ChildItem .xforge/decisions/DR-*.md para listar. cat .xforge/decisions/ADR-INDEX.md para indice.

DR draft sumiu

Causa: Numero duplicado ou em archive/.

Solucao: Get-ChildItem .xforge/decisions/ -Recurse | Where Status -eq draft para encontrar.

Quero editar DR approved

Causa: Impossivel (release-immutability).

Solucao: Crie NOVA DR que referencia a anterior. Use Supersedes: DR-XXXX no frontmatter.

ADR-INDEX nao atualiza

Causa: Voce criou DR mas esqueceu de adicionar ao indice.

Solucao: Edit ADR-INDEX.md e adicione a DR nova. Sem isso, nao aparece no catalogo.
