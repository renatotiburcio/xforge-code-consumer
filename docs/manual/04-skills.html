04 - Skills (172 modulos)

O que e

Skills sao modulos reutilizaveis de conhecimento especializado. Cada skill tem SKILL.md com frontmatter YAML (name, description, type, category, applicabilityScope) + corpo markdown com teoria, exemplos, e quando usar. Diferente de agents (que sao AI entities), skills sao referencia: carregadas no contexto do LLM quando relevante via applicabilityScope ou keyword matching.

Quando usar

  • Quando voce quer conhecimento profundo em um dominio especifico (ex: NFe, EF Core, Blazor)
  • Quando voce quer ensinar um agente novo a usar XForge
  • Quando voce quer criar uma skill nova (ver skill-creator)
  • Quando quer descobrir skills por categoria (registry.json)

Como usar

Sintaxe canonica (uso direto):

# Ver lista de skills
ls .kilo/skills/

# Ler skill especifica
cat .kilo/skills/dotnet-enterprise-expert/SKILL.md

# Criar skill nova
mkdir .kilo/skills/minha-skill
cat > .kilo/skills/minha-skill/SKILL.md <<EOF
---
name: minha-skill
description: O que faz
type: skill
category: Categoria
applicabilityScope: ["dotnet"]
---
# Conteudo...
EOF

# Regenerar registry
regenerate-registries.ps1

Parametros

ParametroTipoDefaultDescricao
namestringobrigatorioIdentificador unico. kebab-case en, max 64 chars.
descriptionstringobrigatorio1-2 frases. Usado para discoverability.
typestring"skill"Fixo. Skills tem type=skill.
categorystring"Geral"Backend, Frontend, Fiscal, Seguranca, etc.
applicabilityScopearray["*"]Stacks compativeis. Router filtra.

Exemplos praticos

Cenario 1: Descobrir skills por stack (.NET)

Contexto: Voce trabalha em projeto .NET 8. Quer saber quais skills carregam.

stack-detector.ps1 --check-scope .kilo/skills/
# Output: 87 skills compativeis com dotnet
# Lista inclui: dotnet-enterprise-expert, efcore-expert, blazor-*,
# automapper-*, xforge-mediatr-*, etc.

# Ver detalhes de uma
Get-Content .kilo/skills/dotnet-enterprise-expert/SKILL.md | Select-Object -First 30

Output esperado (parcial):

---
name: dotnet-enterprise-expert
description: Expert in .NET 10, ASP.NET Core, Minimal APIs, libraries...
type: skill
category: Backend
applicabilityScope: ["dotnet"]
version: 1.0.0
---
# .NET Enterprise Expert
[conteudo...]

Edge case: Se applicabilityScope e' ["*"], carrega em qualquer stack. Use com cuidado.

Cenario 2: Criar skill nova (DR-0182 + skill-creator)

Contexto: Voce implementou padrao de feature flags 5x em projetos diferentes. Quer generalizar como skill.

mkdir .kilo/skills/feature-flags-expert
cat > .kilo/skills/feature-flags-expert/SKILL.md <<EOF
---
name: feature-flags-expert
description: Expert em feature flags (LaunchDarkly, Unleash, custom). Toggle sem deploy, A/B testing, kill switches.
type: skill
category: Padroes
applicabilityScope: ["dotnet", "node", "python", "react"]
version: 1.0.0
---
# Feature Flags Expert
## Quando usar
...
## Como usar
...
EOF

regenerate-registries.ps1
git add .kilo/skills/feature-flags-expert/
git commit -m "feat: add feature-flags-expert skill"

Output esperado:

[regenerate] 173 skills (era 172)
[doctor] 0 errors
[commit] feat: add feature-flags-expert skill

Edge case: Skills muito curtas (< 30 linhas) sao marcadas como debt pelo skill-quality-audit.

Cenario 3: Skill canonica XForge (XForge.MediatR)

Contexto: Voce vai implementar CQRS. XForge tem skill canonica.

# SKILL canonica
cat .kilo/skills/xforge-mediatr-cqrs-expert/SKILL.md

# Comando relacionado
cat .kilo/commands/create-api.md
# (mostra como criar endpoint usando XForge.MediatR)

Output esperado (excerpt da SKILL):

## Regras
- NUNCA usar MediatR oficial. Usar XForge.MediatR.
- Commands e Queries separados em pastas
- Handlers sao classes unicas (SRP)
- Validacao com FluentValidation no pipeline
- Result Pattern para retornos

Edge case: Skill canonica pode ser referenciada em DRs e skills derivadas. Ao atualizar, propague para dependentes.

Cenario 4: Skill por dominio de negocio (fiscal)

Contexto: Voce precisa implementar NFe em projeto .NET. Skills fiscais disponiveis.

ls .kilo/skills/ | grep -i fiscal
# fiscal-tax-expert
# sped-fiscal-expert
# esocial-compliance-expert
# accounting-expert
# labor-law-expert
# health-safety-work-expert
# agronegocio-rural-expert

# Skill recomendada para NFe
Get-Content .kilo/skills/fiscal-tax-expert/SKILL.md | Select-String "NFe" -SimpleMatch

Output esperado:

## NFe
- Schema 4.00 (NT 2025.002)
- SEFAZ contingencia: EPEC, SVC-AN, SVC-RS
- Certificado A1/A3 obrigatorio
- Validacoes: schema, regras de negocio, certificado

Edge case: 7 skills de negocio BR. Combine com knowledge packs (dominios/fiscal/) para profundidade.

Cenario 5: Auditar qualidade das skills (B-090)

Contexto: Voce quer ver quais skills tem conteudo fino (debt).

# Ver relatorio
cat .xforge/reports/skill-quality-audit-2026-06-21.md

# Top 20 skills thin
Select-String "short" .xforge/reports/skill-quality-audit-2026-06-21.md | Select-Object -First 20

Output esperado:

| Skill | Lines | Notes |
| accounting-business-expert | 26 | stub-like |
| business-platform-architect | 26 | stub-like |
| commercial-business-expert | 26 | stub-like |
| ...
| 54 of 171 skills with <50 lines (32%) |

Edge case: Debt reduction e continuo. Rode auditoria mensal, expanda skills thin em PRs incrementais.

Troubleshooting

Skill nao carrega em projeto

Causa: applicabilityScope nao bate com stack detectado.

Solucao: stack-detector.ps1 --check-scope .kilo/skills/ para ver quais carregam. Edite applicabilityScope se a skill deve ser mais inclusiva.

"Registry nao atualiza apos criar skill"

Causa: Esqueca de rodar regenerate-registries.ps1.

Solucao: Sempre rode apos criar/modificar skill. Adicione ao pre-commit hook se quiser automatizar.

"Skill criada mas doctor reclama"

Causa: Frontmatter YAML malformado ou field obrigatorio faltando.

Solucao: Use o template canonico (5 fields: name, description, type, category, applicabilityScope). Valide com python -c "import yaml; yaml.safe_load(open('.kilo/skills/<name>/SKILL.md').read().split('---')[1])".

Skill thin mesmo apos criar

Causa: Adicionou so frontmatter sem corpo substancial.

Solucao: Minimo 50 linhas de conteudo. Secoes: Quando usar, Como usar, Exemplos, Edge cases. Use skill-creator skill como referencia.
