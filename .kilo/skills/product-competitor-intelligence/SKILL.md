# product-competitor-intelligence

## Objective

Transform competitor research into a concrete product plan.

## Sources

The analysis can use:

- local files;
- local documentation;
- screenshots;
- PDFs;
- Excel files;
- GitHub repositories;
- official documentation;
- competitor websites;
- product pages;
- support docs;
- YouTube/transcripts when available;
- app screenshots supplied by the user.

## Modes

- LOCAL_ONLY: use only files and local repository.
- WEB_ONLY: use web/public sources.
- HYBRID: combine local knowledge and web sources.
- OFFICIAL_FIRST: prioritize official docs/sites.
- COMPETITIVE_REVERSE_ENGINEERING: infer features, UX, modules, and gaps from public material.

## Required output

- competitor map;
- feature matrix;
- UX/UI observations;
- domain model inferred;
- modules and screens inferred;
- gaps and opportunities;
- product differentiators;
- MVP proposal;
- roadmap;
- backlog;
- risks;
- implementation plan;
- memory update.
