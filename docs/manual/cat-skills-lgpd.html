Cat-skills LGPD - Privacidade e Protecao de Dados

O que e

Catalogo de skills relacionadas a LGPD (Lei Geral de Protecao de Dados - Lei 13.709/2018) e protecao de dados pessoais. Inclui skills para: implementacao de consentimento, gestao de direitos do titular (acesso, correcao, eliminacao, portabilidade), privacy by design, DPO (Data Protection Officer), adequacao a GDPR (europeia), e compliance com ANPD. Skills canonicas: lgpd-expert, data-protection-lgpd-expert, security-lgpd-expert.

Quando usar

  • Quando precisa implementar coleta de consentimento em formularios
  • Quando esta desenhando fluxo de dados pessoais (banco, log, API)
  • Quando precisa responder a um titular que solicitou acesso/eliminacao dos dados
  • Quando esta fazendo auditoria LGPD do projeto
  • NAO use para legislacao internacional (GDPR tem skills proprias)

Como usar

Skills LGPD sao loaded automaticamente quando o projeto tem dados pessoais. Para carregar manualmente:

# Carregar skill LGPD
/load-skill lgpd-expert
# Ver checklist LGPD
/check-lgpd
# Gerar relatorio de adequacao
/lgpd-audit

Skills LGPD seguem DR-0180 (memory namespace) - so carregam em projetos com dados pessoais BR.

Parametros

ParametroTipoDefaultDescricao

Exemplos praticos

Cenario 1: Coletar consentimento LGPD em formulario

Contexto: Voce tem formulario de cadastro e precisa adequar a LGPD.

Output esperado:

Formulario com consentimento explicito (opt-in) + link para politica. Granularidade: marketing separado do consentimento geral.

Edge case: Pre-tick boxes (ja marcados) sao PROIBIDOS pela LGPD art. 8. Use sempre opt-in.

Cenario 2: Implementar direito de acesso

Contexto: Um usuario solicitou acesso aos dados pessoais armazenados.

GET /api/lgpd/meus-dados
Authorization: Bearer {user-token}

Output esperado:

Resposta em ate 15 dias (prazo legal) com: nome, email, CPF parcial, historico de consentimentos, logs de acesso.

Edge case: Implemente tambem direito de ELIMINACAO (Art. 18 VI) - nao apenas acesso.

Cenario 3: Privacy by Design

Contexto: Voce esta desenhando nova feature que coleta dados.

# Checklist PbD:
# - Apenas dados necessarios (minimizacao)
# - Criptografia at-rest e in-transit
# - Retencao definida
# - Logs de acesso ao dado
# - Anonimizacao para analytics
# - Consentimento granular

Output esperado:

Feature adequa a LGPD antes do deploy. Documenta decisoes no DR.

Edge case: PbD deve estar no design ANTES de codar. Revisao de codigo deve validar privacy.

Cenario 4: Relatar incidente de dados

Contexto: Voce detectou vazamento de dados e precisa notificar ANPD.

# Workflow obrigatorio:
# 1. Imediatamente: isolar sistemas afetados
# 2. Em 2 dias uteis: notificar ANPD via portal
# 3. Em paralelo: notificar titulares afetados
# 4. Documentar: causa, dados expostos, numero de afetados
# 5. Implementar: correcao + prevencao

Output esperado:

Conformidade com Art. 48 LGPD. ANPD analisa e pode aplicar sancao se houver negligencia.

Edge case: Atraso na notificacao = multa adicional. Documente TUDO.

Cenario 5: Adequacao a GDPR (europeia)

Contexto: Sua empresa tem clientes na Europa, precisa adequar tambem a GDPR.

# Diff LGPD vs GDPR:
# LGPD: ANPD fiscaliza (Brasil)
# GDPR: CNIL/ICO fiscaliza (Europa)
# Ambas exigem: consentimento, direito ao esquecimento, DPO
# GDPR tem: privacy by design MAIS rigoroso
# LGPD: DPO obrigatorio para empresas que tratam dados sensiveis em escala

Output esperado:

Implementar pratica mais rigorosa (GDPR) cobre ambas. Use skill data-protection-lgpd-expert que cobre LGPD+GDPR.

Edge case: Considere tambem CCPA (California) em paralelo.

Troubleshooting

Skill LGPD nao carrega no projeto

Causa: DR-0180 (memory namespace) - skill so carrega em projetos com dados pessoais

Solucao: Adicione campo data_classification em .xforge/project-dna/PROJECT-DNA.md. Use classification: personal

Duvida se campo X e dado pessoal

Causa: LGPD define dado pessoal como qualquer info que identifique pessoa (direta ou indiretamente)

Solucao: Em duvida, trate como pessoal. CPF, email, IP, geolocalizacao, cookie ID sao pessoais.

Como obter DPO se empresa pequena

Causa: DPO pode ser terceirizado (pessoa fisica ou juridica)

Solucao: Contrate DPO as a service. Skills incluem templates de contrato e checklist.

Titular pede exclusao mas dados estao em log

Causa: Logs sao necessarios para auditoria/seguranca, mas contem dados pessoais

Solucao: Anonimize logs (substitua CPF por hash, email por user-id). Mantenha log mas remova dado pessoal.
