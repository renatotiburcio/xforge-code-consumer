08 - Knowledge Base (319 entries)

O que e

Knowledge e' a base indexada de 319 entries em .xforge/knowledge/, fonte do RAG. Cada entry tem frontmatter YAML (path, domain, applicabilityScope, trustScore, keywords) + corpo markdown. Router filtra por stack ativo via applicabilityScope (DR-0180). Em projeto .NET medio, 187 de 319 entries carregam (58.7%).

Quando usar

  • Quando voce quer consultar conhecimento especifico (NFe, LGPD, EF Core)
  • Quando quer auditar quais entries carregam no seu projeto
  • Quando quer criar knowledge entry nova
  • Quando quer filtrar por dominio (fiscal, contabil, lgpd)

Como usar

Sintaxe canonica (consulta programatica):

# Ver INDEX.json
cat .xforge/knowledge/INDEX.json | python -m json.tool | head -50

# Filtrar por stack
stack-detector.ps1 --check-scope .xforge/knowledge/INDEX.json

# Listar por dominio
ls .xforge/knowledge/dominios/
# fiscal/ contabil/ trabalhista/ rh/ financeiro/ agro/ comercial/

# Criar entry nova
echo > .xforge/knowledge/dominios/meu-dominio/topico.md
# [frontmatter + corpo]
# Adicionar ao INDEX.json
python .xforge/scripts/rag_local.py reindex

Parametros

ParametroTipoDefaultDescricao
pathstringobrigatorioCaminho relativo a .xforge/knowledge/.
domainstringobrigatorioDominio: fiscal, contabil, lgpd, erp, etc.
applicabilityScopearray["*"]Stacks compativeis. Router filtra.
trustScoreint (0-100)80100=oficial, 90=codigo, 80=blog senior.
keywordsarray[]3-7 termos para busca RAG.
relevancestringmediumhigh, medium, low.

Exemplos praticos

Cenario 1: Knowledge por dominio BR (fiscal)

Contexto: Voce trabalha em projeto ERP fiscal. Quais knowledge entries carregam?

ls .xforge/knowledge/dominios/fiscal/ | head -10
# nfe/basics.md
# sped/efd-icms-ipi.md
# ibs-cbs/reforma.md
# estados-icms-por-uf/details.md
# icms-st/calculo.md

# Ver uma entry
cat .xforge/knowledge/dominios/fiscal/nfe/basics.md

Output esperado (excerpt):

# NFe Basics
- Schema 4.00 (NT 2025.002)
- SEFAZ contingencia: EPEC, SVC-AN, SVC-RS
- Certificado A1/A3 obrigatorio
- Validacoes: schema, regras de negocio, certificado
- Eventos: cancelamento, carta de correcao, manifestacao

Edge case: Entries tem TTL. Reformas (IBS/CBS 2025) tem expiresAt. Rode knowledge-ttl-validation para limpar stale.

Cenario 2: Knowledge universal (LGPD)

Contexto: LGPD aplica a qualquer stack. Use ["*"].

ls .xforge/knowledge/compliance/
# lgpd-basics.md
# lgpd-direitos-titular.md
# lgpd-operacional.md
# privacy-by-design.md
# encarregado-dpo.md
# transferencia-internacional.md
# bases-legais.md
# retencao-documentos.md

# Entry com scope ["*"]
cat .xforge/knowledge/compliance/lgpd-basics.md | head -10
---
domain: lgpd
applicabilityScope: ["*"]
trustScore: 95
source: "LGPL Lei 13.709/2018 + ANPD guidelines"
---

Output esperado:

# LGPD Basics
- Lei 13.709/2018 (vigente desde 2020)
- 10 bases legais para tratamento
- 9 direitos do titular
- DPO obrigatorio
- ANPD pode multar ate 2% do faturamento (max R$ 50M por infracao)
- Privacy by Design (Cavoukian)

Edge case: Use ["*"] com cuidado. Duvida? Escolha 2-3 stacks especificos.

# - dominios/lgpd: 9
# - packs/xforge-efcore: 12

# Ver pattern especifico
cat .xforge/knowledge/padroes/repository-pattern.md

Cenario 4: Criar knowledge entry nova

Contexto: Voce aprendeu um padrao de feature flags. Quer generalizar como knowledge.

# 1. Criar arquivo
cat > .xforge/knowledge/padroes/feature-flags.md <<EOF
---
domain: padroes
applicabilityScope: ["dotnet", "node", "python"]
trustScore: 85
keywords: [feature-flags, toggle, deploy]
relevance: high
---
# Feature Flags
Feature flags permitem toggling de funcionalidades sem deploy...

## Quando usar
- A/B testing
- Kill switches (desabilitar feature em prod)
- Progressive rollout
EOF

# 2. Adicionar ao INDEX.json
# (entry: path, domain, applicabilityScope, trustScore, keywords)

# 3. Reindexar RAG
python .xforge/scripts/rag_local.py reindex
# Output: 748 docs (era 747, +1)

Output esperado:

[reindex] 320 entries (era 319, +1)
[reindex] feature-flags.md: scope=3 stacks, trustScore=85
[done] Knowledge router recarregado

Edge case: Edicao do INDEX.json e manual. Para bulk add, use skill knowledge-ingestion.

Contexto: Voce quer saber quais knowledge entries precisam revalidacao (legislacao mudou).

powershell .kilo/automation/scripts/memory-ttl-validation.ps1

[ttl] 23 entries com TTL > 1 ano (REVALIDAR)

Troubleshooting

Knowledge nao carrega

Causa: applicabilityScope nao bate com stack.

Solucao: stack-detector.ps1 --check-scope .xforge/knowledge/INDEX.json para ver quais carregam.

Entry existe mas INDEX.json nao lista

Causa: RAG reindex rodou mas INDEX.json nao foi atualizado.

Solucao: Edite INDEX.json e adicione entry. RAG chunks.jsonl ja tem (apenas reindex adiciona). Mas router usa INDEX.json para carregar no contexto.

Knowledge stale (legislacao mudou)

Causa: TTL expirado, legislacao mudou (ex: Reforma Tributaria).

Solucao: Use skill curated-operational-knowledge-expert para revalidar. Ou knowledge-ttl-validation para bulk cleanup.

"Knowledge INDEX.json corrompido"

Causa: JSON mal-formado (editor salvou mal, virgula faltando).

Solucao: python -c "import json; json.load(open('.xforge/knowledge/INDEX.json'))" para localizar erro. Backup em INDEX.json.bak-YYYYMMDD-HHMM.
