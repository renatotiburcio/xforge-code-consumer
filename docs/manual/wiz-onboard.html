Wizard /onboard - Onboarding de Novo Dev

O que e

/onboard cria checklist personalizado para novo dev: setup de ambiente, acesso a repos, leitura inicial, primeira task, mentor pairing. Inclui tambem tour guiado pelos 66 genios, 172 skills, 81 agents, 144 commands. Plano salvo em .xforge/onboarding/<nome>/.

Quando usar

  • Quando novo dev entra no time
  • Quando quer acelerar ramp-up
  • Quando quer garantir que novo dev entende template

Como usar

Sintaxe canonica:

/onboard --name "Ana Costa" --role backend --stack dotnet --mentor "Pedro Lima" --duration "3 weeks"

Para invocar: abra o XForge Code CLI e digite o comando. O wizard guia via Q&A interativo.

Exemplos praticos

Cenario 1: Onboarding backend .NET (pos-/forge)

Contexto: Novo dev entrando no time que mantem MeuProjeto (.NET 8 + Blazor). Plano personalizado baseado no stack detectado.

/onboard --name "Ana Costa" --role backend --stack dotnet --mentor "Pedro Lima" --duration "3 weeks"

Output esperado:

[onboard] v50.6.0 - Onboarding Wizard
[detect] Projeto: MeuProjeto (stack: .NET 8 + Blazor Server + EF Core + Identity)
[plan] 47 tasks geradas, 5 milestones, 3 weeks
  [1/47] Setup ambiente (.NET 8 SDK, VS Code, Git, Docker)
  [2/47] Clone MeuProjeto + xforge-init
  [3/47] Build + run local (dotnet run)
  [4/47] Explorar arquitetura: Domain + Application + Infrastructure
  [5/47] Rodar testes (dotnet test)
  [6/47] Entender LGPD setup (DPO config + Identity)
  ...
  [45/47] First PR: adicionar campo Descricao na entidade Produto
  [46/47] Code review com mentor
  [47/47] Merge + deploy staging

[onboard] Plano salvo em .xforge/onboarding/ana-costa/
[onboard] Mentor (Pedro Lima) sera notificado.

[CONTINUIDADE]:
  - Ana pode retomar plano: /onboard --resume
  - Marcos mentoriao marca sessions
  - Primeira task (45) tem starter code gerado por /forge

Edge case: Se stack nao detectado, wizard pede input manual.

Cenario 2: Onboarding fullstack com mentor

Contexto: Dev fullstack entrando no time. Quer parear com senior.

/onboard --name "Joao Silva" --role fullstack --mentor "Maria Santos" --duration "4 weeks"

Output esperado:

[onboard] Plano gerado: 62 tasks, 6 milestones, 4 weeks
  [1-10] Setup + ambiente (dotnet + node + docker)
  [11-20] Explorar backend (Domain, Application, Infrastructure)
  [21-30] Explorar frontend (React/Angular scaffold)
  [31-40] Integracao (API calls, auth flow)
  [41-50] Testes (unit + integration + e2e)
  [51-60] First features (2 PRs)
  [61-62] Review + deploy

[onboard] 3 sessions de pairing incluidas (week 1, 2, 4)
[onboard] Mentor: Maria Santos (notificada)

Edge case: Fullstack gera tasks para backend + frontend + integracao.

Cenario 3: Onboarding QA (testes)

Contexto: QA entrando no time. Foco em testes e qualidade.

/onboard --name "Carlos QA" --role qa --duration "2 weeks"

Output esperado:

[onboard] Plano gerado: 38 tasks, 4 milestones, 2 weeks
  [1-8] Setup + ambiente + ferramentas (Postman, Selenium, k6)
  [9-18] Explorar testes existentes (UnitTests + IntegrationTests)
  [19-28] Criar novos testes (cobertura de gaps)
  [29-38] E2E tests + performance tests + relatorio

[onboard] Foco: cobertura de testes, nao desenvolvimento.

Edge case: QA onboarding nao inclui tasks de feature development.
