11 - FAQ (30 perguntas)

O que e

Perguntas frequentes sobre XForge. Respostas curtas com link para pagina detalhada. Diferente de 08-knowledge (conhecimento profundo), FAQ cobre duvidas operacionais do dia-a-dia.

Quando usar

  • Quando voce tem duvida rapida (1-2 min para resolver)
  • Antes de abrir issue no GitHub
  • Para onboardar novos usuarios
  • NAO use para decisoes arquiteturais (use 03-gcf)

Exemplos praticos

Cenario 1: Geral (5 perguntas)

Contexto: Duvidas gerais sobre XForge.

Q1: O que e' XForge?

Enterprise Development OS: template + runtime + knowledge + manual. NAO e' produto, NAO e' sample. E' o esqueleto que outros projetos usam como ponto de partida.

Q2: Posso usar XForge sem IA?

Sim. Skills, commands e rules funcionam offline. AI e' opcional para GCF (Conselho dos Genios).

Q3: Qual a diferenca entre .kilo/ e .xforge/?

.kilo/ e' operacional (XForgeCode compat 100%, git preservado). .xforge/ e' estado do projeto (decisions, knowledge, memory, scripts).

Q4: Posso deletar .xforge/ depois de clonar?

NUNCA. .xforge/ contem decisoes, knowledge e memoria do projeto. Use xforge reset memory para limpar memoria especifica.

Q5: XForge assume stack?

NAO. Per DR-0180, stack-agnostic. Detecta via sinais. Knowledge entries tem applicabilityScope.

Cenario 2: Stack & Decisoes (5 perguntas)

Contexto: Duvidas de stack e decisao tecnica.

Q6: Devo usar MediatR ou XForge.MediatR?

Sempre XForge.MediatR (github.com/renatotiburcio/XForge.MediatR). MediatR oficial proibido em projetos XForge.

Q7: AutoMapper ou Mapster?

Sempre AutoMapper. Mapster proibido. Validar ultima versao estavel antes de usar.

Q8: PostgreSQL, MySQL ou SQLite?

Producao: PostgreSQL (preferido) ou MySQL. Dev/teste: SQLite ou InMemory. Detalhes em database-efcore-expert skill.

Q9: Quantas skills/agents posso ter?

Skills: ~200 maximo (sem degradar context). Agents: ~100 canonicos. Commands: ~150. Alem disso, considerar sharding.

Q10: Posso rodar XForge em CI/CD?

Sim. Use .\.kilo\automation\scripts\check_loop_discipline.py no pipeline. Veja .github/workflows/engineer-validate.yml.

Cenario 3: GCF (Conselho dos Genios)

Contexto: Duvidas sobre GCF.

Q11: Quando acionar o Conselho?

Decisao nao-trivial: arquitetural, seguranca, UX, produto, refactor, breaking change, bug recorrente, sprint planning. NAO acionar para tarefas triviais.

Q12: Quantos genios existem?

38 originais (AG001-AG038, conceituais) + 28 verticais (AG039-AG066, arquivos) = 66 total.

Q13: Posso adicionar genio novo?

Sim. Use comando CREATE_GENIUS em .kilo/rules/02-genius-council-framework.md secao 10. Resultado: arquivo + DR + update council-members.

Q14: Devil's Advocate (AG999) pode vetar?

Nao diretamente. AG999 levanta objecoes que DEVEM ser respondidas. Guardian Security VETA se FAIL.

Q15: Quanto tempo leva um Conselho completo?

2-5 min para decisao L/XL (7 fases). 30s-1min para M (3 fases + AG999).

Cenario 4: Documentation & DRs

Contexto: Duvidas sobre docs e DRs.

Q16: Onde esta o indice oficial de DRs?

.xforge/decisions/ADR-INDEX.md. 25 DRs ativos, 52 em archive.

Q17: Posso editar DR antiga?

NAO. Per DR-0087 (release-immutability). Crie NOVA DR que referencia a anterior.

Q18: Qual template de DR usar?

3-fases (DR-0087): Problema -> Decisao -> Riscos. Detalhes em 09-decisions.

Q19: Como atualizo o manual?

Edite o HTML canonico em docs/manual/. Mantenha 6 secoes, 1 hero, 1 footer. Use check-manual.ps1 para validar.

Q20: O que e' B-090?

Manual Content Depth Standard (DR-0182): cada pagina deve ter 6 secoes canonicas com 3-7 exemplos, < 15 KB, copy-pasteable. Validador: check-manual-content.ps1.

Cenario 5: Scripts & Performance

Contexto: Duvidas sobre scripts e performance.

Q21: Como rodar doctor?

.\.kilo\automation\scripts\doctor.ps1. Exit 0 = green, exit 1 = blocked.

Q22: O que e' check-manual.ps1?

Validator (DR-0181) que valida: 1 HTML por pagina, nav identica, 0 dead refs, < 500 KB. Sai 0=ok, 1=dirty.

Q23: Como regenerar registries?

regenerate-registries.ps1. Regenera skill/expert/rule/command-registry.json a partir do filesystem. Preserva publicCommands.

Q24: XForge funciona com 16GB RAM?

Sim, mas com Ollama local (LLM) o ideal e' 32 GB. Para XForgeCode cloud, 16 GB basta.

Q25: Como adicionar knowledge entry?

Crie arquivo em .xforge/knowledge/<dominio>/<topico>.md, adicione ao INDEX.json, rode rag_local.py reindex.

Cenario 6: Troubleshooting (5 top errors)

Contexto: Erros comuns.

Q26: "PowerShell execution policy bloqueia script"

Solucao: Set-ExecutionPolicy -Scope CurrentUser RemoteSigned (one-time).

Q27: Knowledge INDEX.json corrompido

Backup em INDEX.json.bak-YYYYMMDD-HHMM. Regenerar com knowledge-validation.ps1.

Q28: Skill nao carrega

Verifique .kilo/core/registries/skill-registry.json. Cada skill precisa de entry + frontmatter correto.

Q29: "xforge-init" falha em Windows

Verifique PowerShell 5.1+, ExecutionPolicy. Use pwsh (PowerShell Core) se disponivel.

Q30: Como reporto bug?

github.com/renatotiburcio/xforge/issues. Use template de bug report com stack, doctor output, repro steps.
