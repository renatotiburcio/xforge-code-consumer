Wizard /domain - Modelar Dominio DDD

O que e

/domain guia modelagem DDD completa: identifica contextos delimitados (bounded contexts), agregados, value objects, domain events, e gera estrutura inicial de pastas + classes base. Usa --ddd-style tactical (padrao) ou strategic (context mapping).

Quando usar

  • Quando comeca projeto novo e precisa modelar dominio
  • Quando projeto legado precisa de refactor para DDD
  • Quando quer documentar dominio existente para o time

Como usar

Sintaxe canonica:

/domain --project MeuProjeto --bounded-contexts 5 --ddd-style both --events

Para invocar: abra o XForge Code CLI e digite o comando. O wizard guia via Q&A interativo.

Exemplos praticos

Cenario 1: Modelar dominio ERP (pos-/forge)

Contexto: Apos rodar /forge new MeuProjeto --patterns ddd, voce quer modelar o dominio completo com bounded contexts, agregados e events.

/domain --project MeuProjeto --bounded-contexts 5 --ddd-style both --events

Output esperado:

[domain] Analisando 47 entidades do scaffold...
[domain] 5 bounded contexts identificados:
  1. Cadastros (Customer, Supplier, Product)
  2. Vendas (Order, OrderItem, Payment)
  3. Estoque (Stock, Movement, Warehouse)
  4. Fiscal (Invoice, Tax, NFe)
  5. Identity (User, Role, Permission)
[domain] 12 Aggregates + 23 ValueObjects + 41 DomainEvents gerados
[domain] Context Map: docs/domain/context-map.md
[domain] Total: 78 arquivos em src/Domain/

Edge case: Se < 3 bounded contexts, wizard sugere monolito modular (sem separacao DDD).

Cenario 2: Refactor de projeto legado para DDD

Contexto: Projeto existente com 200+ classes em pasta unica. Quer extrair bounded contexts.

/domain --project Legado --ddd-style strategic --output ./domain-analysis/

Output esperado:

[domain] Analisando 214 classes em src/Legacy/...
[domain] 8 bounded contexts detectados (por namespace + using analysis)
[domain] Context Map com 12 relacoes entre contexts
[domain] Anti-Corruption Layer sugerido: 3 contexts
[domain] Relatorio: ./domain-analysis/ddd-report.md

Edge case: Analise estatica pode detectar contexts errados; revise o context-map.md antes de implementar.

Cenario 3: Documentar dominio existente (docs)

Contexto: Time precisa de documentacao compartilhada do dominio. Quer gerar docs automaticamente.

/domain --project MeuProjeto --output docs/domain/ --language portuguese

Output esperado:

[domain] Gerando documentacao em pt-BR...
[domain] docs/domain/context-map.md
[domain] docs/domain/bounded-contexts/cadastros.md
[domain] docs/domain/bounded-contexts/vendas.md
[domain] docs/domain/aggregates/customer-aggregate.md
[domain] docs/domain/value-objects/email.md
[domain] docs/domain/events/customer-created.md
[domain] Total: 34 docs gerados.

Edge case: Use --language portuguese para docs em pt-BR (default: en).
