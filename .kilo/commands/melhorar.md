# /melhorar

## Objetivo
Melhorar continuamente o sistema: refatoracao, performance, UX, arquitetura e auto-melhoria.

## Sub-comandos

| Sub-comando | Acao |
|-------------|------|
| `/melhorar codigo` | Refatorar e limpar codigo |
| `/melhorar arquitetura` | Melhorar design arquitetural |
| `/melhorar UX` | Melhorar experiencia do usuario |
| `/melhorar performance` | Otimizar performance |
| `/melhorar auto` | Auto-melhoria baseada em feedback |

## Procedimento

### Codigo
1. Identificar code smells
2. Aplicar Clean Code / SOLID
3. Verificar testes

### Arquitetura
1. Identificar violacoes de DDD/Clean Architecture
2. Propor melhorias
3. Criar DR se necessario

### UX
1. Identificar pontos de atrito
2. Simplificar fluxos
3. Melhorar feedback visual

### Performance
1. Identificar gargalos
2. Otimizar queries
3. Implementar caching

### Auto-melhoria
1. Ler feedback log
2. Identificar padroes de erro
3. Criar regras preventivas

## Uso
```
/melhorar codigo
/melhorar arquitetura
/melhorar UX
/melhorar performance
/melhorar auto
```
