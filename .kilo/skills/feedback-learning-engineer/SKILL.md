# feedback-learning-engineer

## Objetivo

Capturar, classificar e transformar feedback em ações de melhoria.

## Fontes de Feedback

| Fonte | Tipo | Prioridade |
|-------|------|:----------:|
| Usuários finais | UX, bugs | Alta |
| Stakeholders | Features, roadmap | Alta |
| Time de suporte | Issues recorrentes | Média |
| Code review | Qualidade | Média |
| Métricas | Performance | Média |

## Classificação

| Categoria | Ação |
|-----------|------|
| **Bug** | Criar issue, priorizar |
| **Feature request** | Adicionar ao backlog |
| **UX improvement** | Design review |
| **Performance** | Benchmark, otimizar |
| **Documentação** | Atualizar docs |
| **Processo** | Melhorar workflow |

## Procedimento

### 1. Capturar
- Formato padronizado
- Fonte e contexto
- Severidade e impacto

### 2. Classificar
- Categoria
- Prioridade
- Componente afetado

### 3. Analisar
- Frequência (recorrente ou pontual)
- Impacto (muitos ou poucos afetados)
- Esforço de correção

### 4. Ação
- Criar issue/backlog item
- Designar responsável
- Definir deadline
- Acompanhar conclusão

## Regras

- Todo feedback deve ter resposta
- Feedback recorrente → prioridade alta
- Documentar padrões identificados
- Medir satisfação pós-melhoria
